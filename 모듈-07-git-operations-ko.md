# 모듈 7: Git 조작 및 버전 관리 (Version Control)

**목표:** Claude Code의 막강한 서포트를 받아 Git 워크플로우를 자유자재로 다루기

**예상 소요 시간:** 40-50분

---

## 무엇을 배우게 되나요?

이 모듈에서는 단순 Commit(커밋), Branch(브랜치) 분기부터 Pull Request(PR) 생성, 그리고 끔찍한 Merge Conflict(충돌) 해결까지 Git 시스템을 Claude Code를 통해 어떻게 터뜨리는지 배웁니다. 살면서 "자잘한 버그 수정", "이것저것 건드림" 같은 쓰레기 커밋 메시지를 적어본 적이 있다면 이 모듈이 당신을 구원할 겁니다.

---

## 레슨 1: 버전 관리가 왜 중요한가요?

### 버전 관리(Version Control)란?

시간의 흐름에 따라 내 코드가 어떻게 변했는지 그 기록을 모두 다 추적/보관하는 역사책입니다.

- **무한 되돌리기(Undo)** -- 코드를 날려먹어도 과거 어느 시점이든 완벽 복구 가능
- **현대적 협업** -- 멱살 잡지 않고 다른 사람들과 안전하게 한 프로젝트를 쪼개서 개발
- **히스토리** -- 언제, 누가, 도대체 왜 이걸 이따위로 고쳐놨는지 범인 색출 가능
- **안전망** -- "이거 건드리면 좆될까?" 하는 두려움 없이 과감한 실험 가능

### Git 핵심 요약

- **저장소 (Repo)** -- Git이 지켜보고 있는 프로젝트 폴더 전체. 내 컴퓨터에 있으면 Local, Github 에 올려두면 Remote.
- **Commit (커밋)** -- 특정 시점의 내 코드를 "찰칵" 사진으로 찍어 남기는 박제 행위. 항상 "무엇을 바꿨다"라는 이름표(메시지)를 달아야 합니다.
- **Branch (브랜치)** -- 평행 우주. 원본 코드(Main)는 냅두고, 혼자 빠져나와서 뚝딱뚝딱 지지고 볶는 기능 작업용 가지. 다 만들고 작동 잘 되면 원본에(Merge) 섞습니다.

## 레슨 2: Claude Code로 기초 Git 작업 다루기

### 현재 꼬라지 파악하기 (Status)

지금 내 코드가 뭐 얼마나 바뀌었는지 볼 때:

```
지금 git status 상태 좀 싹 보여줘
내가 방금까지 뭔 파일들을 수정했었지?
```

Claude Code는 뒤에서 `git status`를 치고 결과를 싹 브리핑해 줍니다 (빨간색, 초록색 등으로 추가된 놈 삭제된 놈 다 보여줌).

---

### 바뀐 코드 내역 확인하기 (Diff)

정확히 코드 어느 라인을 어떻게 조져놨는지 볼 때:

```
내가 만들어 논 git diff 내용들 좀 까서 보여줘
auth.js 파일 수정본이랑 예전 거랑 어떻게 다른지 비교해 봐
```

Claude Code는 옛날 라인은 빨갛게 (-), 추가한 라인은 초록색 (+) 으로 기가 막히게 시각화 교차 검증을 해줍니다.

---

### Claude Code로 "예술적인" 커밋(Commit) 생성하기

이게 진짜 Claude Code를 쓰면서 환호성을 지르게 되는 순간입니다.

**원시인들의 옛날 짓거리:**

```bash
git add .
git commit -m "버그 이것저것 픽스함"  # 진짜 최악의 범죄
```

**우아한 Claude Code 활용법:**

```
내가 지금까지 작업/수정한 파일들 죄다 모아서 커밋 하나 올려 줘
```

이렇게 던지면, Claude Code는 제가 친 명령을 바탕으로 무려 당신이 수정한 모든 파일의 소스를 자기가 스캔합니다. 그리고 **당신조차 까먹은 수정 의도를 기가 막힌 작문력으로 뽑아내 커밋 메시지로 포장해버립니다.**

**Claude Code가 대신 써주는 실제 커밋 메시지 예시:**

```
JWT 토큰 기반의 유저 로그인 인증 기능 추가

- 로그인 및 계정 가입 API 엔드포인트 구현 완료
- 비밀번호 보안 처리를 위해 bcrypt 패키지 추가 
- 접근 제한이 필요한 라우트를 막기 위해 JWT 판독 미들웨어 설계
- 데이터베이스 모델에 User 스키마 안착
- Auth 과정에서 패스워드나 이메일 빈칸 못 들어오게 Validation 방어막 추가

이 커밋은 시스템 전체 API에 최상위 보안 유저 인증 수단을 제공하기 위함입니다.
```

이제 "이것저것 건드림" 같은 부끄러운 커밋 메시지를 쓸 일이 지구상에서 사라졌습니다.

## 레슨 3: 브랜치 (Branches) 자유롭게 찢기

### 브랜치를 왜 써야 합니까?

원본 코드(Main) 망가뜨릴 걱정 없이 격리된 수술실에서 암 덩어리를 고치거나, 남들과 충돌 안 나게 나만의 방에서 평행으로 뚝딱뚝딱 기능 짜고 나중에 합치기(Merge) 위해 씁니다. 꼭 쓰세요. 진짜 무조건 평생 써야 합니다.

### Claude Code로 브랜치 창조하기

```
feature/user-dashboard 라는 이름으로 브랜치 하나 파서 거기로 넘어가
```

이러면 내부에서 `git checkout -b feature/user-dashboard` 를 실행해 줍니다.

아예 AI 한테 작명(Naming)을 맡겨도 됩니다:

```
나 지금부터 '비밀번호 재설정(Password reset)' 기능 만들 건데 이거에 쓸 알맞은 브랜치 하나 니가 알아서 예쁘게 파서 생성해.
```

### 브랜치 작명 기본 룰 (컨벤션)

- `feature/새기능이름` -- 뭔가 멋진 새 기능 만들 때
- `bugfix/버그내용` -- 버그 잡아 족칠 때
- `hotfix/치명적수정` -- 서버 터지기 일보 직전 응급 수술
- `refactor/리팩토링주제` -- 코드 다이어트 및 구조 잡을 때

### 브랜치 와리가리 춤추기 (Switching)

```
본진(main branch)으로 다시 넘어가 줘
방금 팠던 내 기능 연습 브랜치로 다시 빠져
```

## 레슨 4: 풀 리퀘스트 (PR, Pull Requests) 벼락치기로 찍어내기

### PR (Pull Request) 이 뭔가요?

당신이 외딴 브랜치에서 만든 쩌는 코드를 본진(Main)에 합쳐달라고 팀원 원로 위원회에 "승인 요청"을 때리는 결재 서류입니다.

### Claude Code 로 풀 리퀘스트 날리기

**진짜 이거 하나를 위해 Claude Code에 돈 낼 가치가 차고 넘칩니다.** PR 서류에 내가 무슨 짓을 했는지 하나하나 설명하는 건 고역입니다.

```
내가 지금 브랜치에서 작업한 내용들 싹 다 Github에 Pull Request로 쏴 줘
```

**Claude Code의 미친 퍼포먼스:**

1. 당신 브랜치의 모든 커밋 기록 스캔
2. 수정된 모든 코드 라인 분석
3. PR 본문 제목, 요약본, 리스트, 심지어 '테스트 완료해야 하는 체크리스트' 구이(Markdown) 양식까지 완벽히 글짓기 해냄.
4. Github CLI 연동으로 곧바로 Github.com 에 PR 슛을 꽂아 넣음.

*(아까 모듈 7 본문에 적힌 엄청나게 길고 아름다운 마크다운 PR 예시 문서가 단 5초 만에 떨어집니다.)*

## 레슨 5: 커밋 버튼 누르기 전, AI의 코드 감리(Review) 받기

아무 생각 없이 커밋 때리기 전에, 클로드에게 마지막 내 코드 상태를 점검(Review)받는 깐깐한 버릇을 들이세요. 진짜 X팔린 실수를 막아줍니다.

**1. 나한테 먼저 브리핑해**

```
내가 커밋 때리기 직전인데, 지금까지 바뀐 거 한 번 화면에 싹 나열해 봐
```

**2. 꼬투리 좀 잡아봐**

```
내가 짜 놓은 이 새 코드에 허점 없는지 리뷰 좀 빡세게 해:
- 보안 뚫릴 구석은 없는지
- 코드 더럽게 짠 건 없는지
- 로직 에러나 에러 핸들링 빼먹은 건 없는지
```

**3. 지적 사항 수정해**

```
오, 로그인 로직 쪽 보안 빵꾸난 거 네가 지적한 대로 코딩 고쳐
```

**4. 결재 승인**

```
수고했어. 이제 완벽하니까 방금 변경사항들 다 모아서 커밋 하나 올려.
```

## 레슨 6: 최악의 악몽, 머지 컨플릭트(Merge Conflicts) 해결사

### Merge Conflicts (충돌) 이 뭔가요?

내 브랜치랑 Main 브랜치를 퓨전(Merge)하려는데, 하필 내가 수정한 소스코드 '13번째 줄'을 다른 미친 팀원놈이 이미 먼저 건드려서 Main에 끼워 넣어놓은 겁니다. Git은 도대체 누구 코드가 정답인지 알 수 없어서 에러를 뿜으며 수동으로 사람이 선택하라고 똥을 던집니다.

### Claude Code로 병목 돌파하기

충돌 터져서 멍때릴 때 살려주세요 라고 외칩니다:

```
지금 server.js 파일에서 머지 컨플릭트 에러 떴다!! 이거 어떻게 합쳐야 하는지 좀 해결해 줘!!
```

**이러면 Claude 둘이 나타나 대화하듯 해결책을 물어봅니다:**

```
Claude Code: 지금 server.js 옵션 컨플릭트 떴습니다.
- HEAD (Main 정식 버전): PORT 번호를 3000 으로 해놓음
- 당신 브랜치: PORT 번호를 8080 으로 치환함

어떤 옵션을 살릴까요? 아니면 환경 변수로 아예 유동적으로 빠지게 제가 다 뜯어고칠까요?

당신: 오 환경 변수 좋네 그걸로 합쳐!
```

이 고통스러운 붉은 글씨 컨플릭트 정리가 AI 한마디로 자동 병합됩니다.

## 레슨 7: Github 명령어도 터미널에서 쏴버리기 (Github CLI)

굳이 마우스나 크롬 브라우저를 켜지 않아도 Claude Code가 내 터미널에서 Github.com 사이트의 모든 기능을 컨트롤(gh) 합니다.

**PR 현황보기**

```
지금 Github에 열려 있는 PR 껍데기들 리스트 다 불러와
```

**이슈(Issue) 꽂기**

```
방금 발견한 버그 Github에 Issue 티켓 하나 발권해서 꽂아:
제목: 폰트 깨짐 현상
내용: 블라블라~
```

**PR 합격 확인 / 병합(Merge) 때리기**

```
PR #42번 깃허브 액션 테스트 통과 잘 떴어? 떴으면 그대로 Merge 버튼 눌러 줘
```

## 모듈 7 요점 체크리스트

모듈 8로 넘어가기 전 점검:

- [ ] 내 코드가 뭐가 변했는지 상태를 점검할 수 있다.
- [ ] 쓰레기 같은 메시지가 아니라, AI가 영혼을 담아 쓴 완벽한 커밋 메시지를 생성하게 지시할 수 있다.
- [ ] 필요할 때마다 방(Branch)을 갈라 파서 이동하며 작업할 줄 안다.
- [ ] 5초 만에 완벽한 스펙이 담긴 PR 제안서를 뚝딱 써서 던질 수 있다.
- [ ] 커밋 전에 코드를 AI에게 결재(Review)받아 멍청한 실수를 차단할 줄 안다.
- [ ] 피눈물 나는 Merge Conflict 충돌을 AI가 해석하고 봉합하도록 조종할 줄 안다.

## 당신을 살려줄 명심보감 Git 룰 (Do's & Don'ts)

**잘하는 짓:**

- 돌아가는 정상 코드만 남긴다.
- Claude Code의 문장력을 빌려 예쁜 커밋 메시지를 쓴다.
- 본진(Main) 건드리지 말고 항상 Branch 방 따로 파서 일한다.

**미친 짓 (절대 금지):**

- 에러 뿜는 쓰레기 코드를 일단 푸시(커밋) 해버린다.
- AWS API 비밀키, DB 접속 비밀번호 써놓고 그대로 Git에 올린다 (진짜 당신 빚쟁이 돼서 자살할 수도 있습니다).
