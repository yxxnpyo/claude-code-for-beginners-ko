## 엔진 시동 및 폭파

### Claude Code 켜기

```bash
claude
```

### 특정 폴더에서 켜기

```bash
cd /path/to/project
claude
```

### 끄기 (퇴근)

```

```

또는 `Ctrl+C`

## 밥줄 (핵심 슬래시 명령어)

| 명령어 | 도대체 뭐하는 거임? |
|---------|-------------|
| `/exit` | Claude Code 종료 |
| `/help` | 도움말 설명서 펼치기 |
| `/clear` | 지금껏 떠든 채팅 기록 싹 지우기 (초기화) |
| `/tasks` | 뒤에서 몰래 도는 백그라운드 작업들 감시하기 |

## 마법의 프롬프트 주문서 (복붙용)

### 1. 파일 조작의 연금술

**새로운 거 창조할 때:**

```
[파일이름] 하나 새로 만들어. 기능은 [무슨 동작하는지] 이거야.
```

**그냥 들여다볼 때:**

```
저기 [파일이름] 좀 열어서 화면에 까봐.
[파일이름] 안에 무슨 내용인지 다 말해.
```

**뜯어고칠 때 (수술):**

```
[파일이름] 열어서 저기 [A] 부분을 [B] 로 바꿔치기 해.
[파일이름] 에 [새로운 기능] 추가해서 끼워 놔.
이거 [파일이름] 에서 나는 버그/에러 당장 잡아 고쳐!
```

**사냥개 풀기:**

```
여기 모든 [파이썬/JS] 파일 확장자 싹 다 뒤져서 목록 뽑아.
[특정 글자 혹은 변수명] 들어간 파일들 좀 죄다 색출해 줘.
내 테스트 코드 파일들 리스트업 해봐.
```

---

### 2. 코드 폭행 (조작/생성/파괴)

**코드 분석 짬처리:**

```
이거 [파일이름/함수이름] 대체 무슨 외계어로 짠 건지 좀 초딩 입장에서 설명해.
이 프로젝트에서 [특정 기능]이 어떻게 굴러가는지 로직 구조 파악해 봐.
```

**새 부품 찍어내기:**

```
[클래스/함수] 하나 새로 작성해 봐. 역할은 [기능 설명] 이야.
```

**버그 사냥 (디버깅):**

```
이 좆같은 버그 좀 잡아 당장 [파일이름] 해부해.
[이런 상황] 일 때 앱이 자꾸 뻗는데(크래시), 어디서 터진 건지 디버그(조사) 좀.
```

**코드 다이어트 (리팩토링):**

```
이 쓰레기 코드 [파일/부분] 을 [이런 디자인 패턴] 써서 예쁘고 깔끔하게 싹 개조해.
코드 좀 알아볼 수 있게 우아하게 리팩토링해 봐.
```

---

### 3. 프로젝트 건설 (초기화 및 실행)

**첫 삽 뜨기:**

```
[프레임워크/언어] 써서 완전 새 프로젝트 [이름] 빈 폴더에 초기화해. 
여기에 추가로 [기능/라이브러리명] 도 싹 다 넣어서 세팅하고!
```

**패키지(의존성) 물어오기:**

```
[패키지명] npm/pip 으로 깔아 놔.
이 프로젝트 구동에 필요한 모듈들 죄다 인스톨 돌려!
```

**터미널에서 손 떼기:**

```
앱 실행해.
개발 서버(Dev Server) 켜 놔.
테스트 봇 돌려.
프로덕션용 빌드(Build) 돌려.
```

### 4. Git 대리 운전

**현황 보고:**

```
지금 git status 상태 좀 읊어.
내가 지금까지 뭐뭐 수정했지? 
git diff 찍어서 뭐가 바뀌었는지 좀 나한테 보고해 줘.
```

**커밋(Commit) 결재 올리기:**

```
본진 저장소 건드리지 말고 방금 바꾼 거 모아서 커밋해. 
메시지는 네가 영혼 갈아서 예쁘게 작문해 봐.
```

**새로 방 파기 (Branch):**

```
[내 브랜치명] 이라는 브랜치 하나 파고 거기로 넘어가자.
다시 [메인 브랜치명] 본진으로 복귀시켜.
```

## 도구(Tool) 멱살 잡고 억지로 부리기

클로드는 보통 알아서 툴을 씁니다. 하지만 좀 답답할 때 강제로 도구를 꺼내라고 명령할 수 있습니다.

**진단 현미경 (Read):**

```
저 [파일이름] Read 툴 써서 읽고 여기 [특정 구역] 만 화면에 읊어.
```

**볼펜 (Write):**

```
새로 Write 해서 [파일이름] 창조하고 저기 [내용] 좀 우려 넣어.
```

**지우개 (Edit):**

```
Edit 꺼내서 [파일이름] 특정 부분만 수술해.
```

**탐지견 (Grep/Glob):**

```
Grep 써서 모든 파일에서 [특수 문자열] 무조건 찌르고 다 찾아내.
Glob 돌려서 여기 *.js 파일 목록 나한테 바쳐.
```

**터미널 조종간 (Bash):**

```
Bash 로 저기 `[명령어]` 좀 터미널에 때려 박아.
```

## 하지 말아야 할 미친 짓 (Do/Don'ts)

**착한 짓:**

- 뭘 해달라고 할 때 구체적으로 쪼아대라.
- 이 프로젝트 전체 문맥(Context)을 앞단에 설명해라.
- `git diff` 돌려서 얘가 뭘 수정했는지 꼭 눈으로 검열하고 넘어가라.
- 버전 관리(Git)는 살길이다, 무조건 커밋 잔뜩 해놔라.

**미친 짓:**

- "게시판 하나 만들어줘" 처럼 뜬구름 잡는 외계어 명령 금지.
- 왕창 한 번에 수정해놓고 중간중간 테스트 안 돌려보기 금지.
- 무지성으로 클로드가 써주는 코드 다 오케이(Yes) 누르기 금지.
- 에러 메시지 떴는데 그 줄 안 읽고 그냥 무시하고 넘어가기 절대 금지!

## 5분 대기 디버깅 (응급실 조치법)

앱 터졌습니까? 당장 이렇게 말하세요:

1. **에러 원문 통역:** `이 에러가 뭔 소리냐 초딩말로 읊어.`
2. **환부 보여주기:** `저 파일 [X]번째 줄 근처 한 번 까서 자세히 좀 봐라.`
3. **비교 대조:** `[X] 이 새끼 원래 딴 데선 잘 됐는데 거기랑 뭐가 다른지 찾아내.`
4. **패키지 점검:** `모듈은 다 똑바로 깔려있긴 하냐?`
5. **격리 병동 테스트:** `요 터진 기능만 쏙 빼서 격리한 미니 테스트 하나 만들어 봐.`
6. **SOS (포기):** `나 못하겠다. 네가 이거 디버깅 싹 다 맡아서 해결 좀 해줘.`
