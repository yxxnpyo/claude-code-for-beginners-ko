# 모듈 1: Claude Code에 오신 것을 환영합니다 - 첫 걸음

**목표:** Claude Code에 익숙해지고 무엇을 할 수 있는지 이해하기

**예상 소요 시간:** 20-30분

---

## 무엇을 배우게 되나요?

이 모듈에서는 가장 기초적인 내용을 다룹니다: Claude Code가 정확히 무엇인지, 어떻게 설치하는지, 어떻게 대화하는지, 그리고 이 과정에서 마주치게 될 전문 용어들에 대해 알아봅니다. 또한, 전체 워크플로우가 작동하는 방식을 볼 수 있도록 작은 Python 스크립트도 함께 만들어 볼 것입니다.

---

## 레슨 1: "AI 페어 프로그래밍"이란 무엇인가요?

### 간단한 설명

AI 페어 프로그래밍이란 원하는 것을 설명하면 AI가 그것을 함께 만들어주는 방식을 말합니다. 프로젝트의 방향을 결정하는 것은 여전히 나(사용자)의 몫이지만, 코드를 작성하고 파일을 읽고 터미널 명령을 실행하고 상황을 설명해 주는 지식 있는 파트너를 옆에 두고 있는 것과 같습니다.

기존의 방식과 비교해 볼까요:

**기존 방식:** 코드를 한 줄 한 줄 직접 입력하고, 공식 문서를 뒤지고, 버그를 수동으로 찾고, 문법과 API를 모두 머릿속에 기억해야 합니다. 작은 오타 하나 때문에 1시간 이상 날리기도 합니다.

**Claude Code 방식:** "포트 3000에서 시작되는 HTTP 서버를 만들어줘"라고 말하면 AI가 코드를 작성하고, 파일을 만들고, 심지어 코드가 잘 돌아가도록 실행까지 해줍니다. 무언가 문제가 생기면 나와 함께 버그를 찾고 해결합니다.

### 간단한 예시 비교

기존이라면 이렇게 했을 겁니다:

```bash
# 구글에 "node.js에서 http 서버 만드는 법" 검색
# 문서 읽기
# 예제 코드 복사/붙여넣기
# 왜 서버가 작동안하는지 디버깅
# 오타 검사
# 수동으로 코드 다시 동작 테스트
```

이제는 이렇게 입력하면 끝입니다:
> "Node.js를 사용해서 3000번 포트에서 'Hello World'를 반환하는 간단한 HTTP 서버를 만들어줘"

그러면 Claude Code가 해당 파일을 생성하고, 코드를 기록하고, 자신이 어떤 작업을 했는지 보고해주고, 실행까지 해줍니다. 그게 끝입니다.

## 레슨 2: Claude Code 설치하기

### 필요한 준비물

설치 전 필요한 환경:

- 운영체제가 macOS, Linux, 혹은 Windows (WSL 환경)인 컴퓨터
- 인터넷 연결
- 터미널이나 커맨드 라인 프로그램
- Claude API 발급 키 (아래에서 받을 예정)

Windows를 쓰신다면 WSL (Windows Subsystem for Linux) 환경이 필요합니다. 복잡하게 들리지만, 사실 간단한 설정으로 바로 적용할 수 있습니다.

### 단계별 설치 과정

#### 1단계: Node.js 설치 (없는 경우)

Claude Code는 Node.js 버전 18 이상이 필요합니다.

이미 설치되어 있는지 확인해 보세요:

```bash
node --version
```

`v18.x.x` 이상의 숫자가 보인다면 완벽합니다. 그렇지 않다면:

1. [nodejs.org](https://nodejs.org) 에 접속합니다.
2. LTS (Long Term Support) 버전을 다운로드합니다.
3. 인스톨러를 실행해 안내에 따라 설치를 마칩니다.

#### 2단계: Claude API 키 받기

1. [console.anthropic.com](https://console.anthropic.com) 접속.
2. 회원가입 또는 로그인합니다.
3. "API Keys" 메뉴로 이동.
4. "Create Key" 버튼을 클릭합니다.
5. 발급된 API 키 복사 (`sk-ant-...` 로 시작하는 문자열).
6. 복사된 키를 안전한 장소에 비밀번호처럼 보관하세요. 타인에게 공유하거나 소스 코드 Git에 실수로라도 올리지 마세요.

#### 3단계: Claude Code 설치

내 컴퓨터의 터미널을 열고 다음 명령어를 입력합니다:

```bash
npm install -g claude-code
```

위 명령어의 간단한 의미:

- `npm` = Node 패키지 매니저 (소프트웨어 다운/설치 프로그램)
- `install` = 설치해라
- `-g` = 전역적으로 (컴퓨터 시스템의 모든 경로 어디에서나 쓸 수 있게)
- `claude-code` = 패키지 이름

진행 상태 표시가 완료될 때까지 잠시 기다립니다.

#### 4단계: 내 API 키 등록하기

설치한 Claude Code에 내 API 키 정보를 입력해 주어야 합니다. 두 가지 방법이 있습니다:

**방식 A: 환경 변수로 저장 (추천)**

macOS/Linux 시스템:

```bash
export ANTHROPIC_API_KEY="나의-api-키-입력"
```

Windows 시스템 (PowerShell):

```powershell
$env:ANTHROPIC_API_KEY="나의-api-키-입력"
```

영구적으로 저장을 원한다면 환경 변수 설정 파일(`~/.bashrc`, `~/.zshrc` 등)에 넣어두세요.

**방식 B: 터미널 설정(Config) 명령어에 바로 입력**

```bash
claude config set apiKey "나의-api-키-입력"
```

#### 5단계: 설치 최종 확인

```bash
claude --version
```

설치된 버전을 알려주는 숫자가 보이면 성공입니다. 확인이 안 되거나 에러가 생기면 터미널 창을 껐다 켜보고 다시 실행해 보세요.

## 레슨 3: Claude Code 인터페이스의 이해

### CLI가 무엇인가요?

CLI (Command Line Interface)는 컴퓨터 프로그램과 텍스트(글) 기반으로 소통하고 조작하는 방식을 뜻합니다. 마우스로 특정 버튼을 꾹 누르는 대신, 명령어를 텍스트 키보드로 타이핑해서 프로그램을 조종합니다. 하지만 엄청 어려운 것은 아니고, 사람과 수다를 떨 듯이 그냥 평상시 언어로 말을 걸면 알아서 다 해줍니다.

### 처음 마주한 화면의 모습

터미널에서 Claude Code를 불러봅시다:

```bash
claude
```

그러면 보통 이런 화면이 뜹니다:

```
╭───────────────────────────────────────────────────╮
│                                                   │
│   Claude Code v1.0.0                             │
│   AI-powered development assistant               │
│                                                   │
╰───────────────────────────────────────────────────╯

Working directory: /home/user/projects

How can I help you today?
>
```

저기 보이는 기호 `>` 옆이 내가 타자를 치는 란입니다. 여기가 나와 Claude Code의 '대화방' 인 셈입니다.

### 화면 요소들의 의미

- **헤더부 (Header)** -- 버전 정보, 현재 활성화된(내가 있는) 폴더 경로, 상태 등을 안내.
- **프롬프트 기호 (`>`)** -- 이 뒤쪽에 내가 일상적인 보통 언어(자연어)로 원하는 것을 적습니다.
- **결과 노출 공간** -- 사용자의 입력 칸 기호 바로 위쪽 여백 공간으로 Claude의 응답, 어떤 기능 사용, 결과물이 위에서 아래로 계속 떨어져 내립니다.

### 직접 해보기

프롬프트 공간에 아무렇게나 말을 걸어보세요:

```
Hello! Can you explain what you can do?
(안녕! 넌 뭘 할 수 있는지 설명해줄래?)
```

Enter 키를 누르세요. 마치 친절한 직장 동료처럼 자기소개와 함께 자기가 가능한 핵심 기능을 상세히 답변할 것입니다.

## 레슨 4: 핵심 용어정리

앞으로 일상처럼 계속 튀어나올 중요 단어들입니다. 무리해서 외울 필요 없습니다. 진도를 빼다가 헷갈리면 여기 돌아와서 가볍게 눈으로 읽으면 됩니다.

**Session (세션)** -- 한 번 Claude Code를 켜서 끄기 전까지의 전체 흐름/대화 기간입니다. Claude를 켜고, 내 할 일을 하고, 껐습니다. 이걸 하나의 '세션'이라고 부릅니다.

**Working Directory (작업 디렉토리)** -- 지금 Claude가 구동 중인 소스, 폴더 경로의 공간. `/home/user/my-project` 와 같은 경로에서 호출되면 그 디렉토리의 파일들을 탐색, 추가하고 조종합니다.

**Tool (도구)** -- 파일 읽기, 쓰기 명령 실행, 수정 같은 Claude Code가 내릴 수 있는 특별 능력치/도구들을 의미합니다. 뭔가 조작을 수행할 때마다 화면에 어떤 행동 도구를 썼는지 안내합니다.

**Prompt (프롬프트)** -- 당신의 주문서나 명령문. "파이썬 언어로 Hello World 화면에 뿌려주는 파일 이름 좀 만들어 줘" 같은 문장 자체.

**Tool Use / Tool Call (도구 사용 / 도구 호출)** -- Claude가 자기가 갖고 있는 도구를 꺼내 쓰는 작업을 말합니다. 가령 "Write tool을 사용함", "Bash tool을 사용함" 처럼 안내가 뜹니다.

**Agent (에이전트)** -- 좀 복잡한 미션을 맡기기 위해 전문화된 도우미/부하직원들. 가령 어떤 폴더 내에서 거대한 소스 코드를 스캔 및 파악하려 할 때, 탐색 특화 임무(Explore) 같은 것을 넘겨받는 부서별 담당 특화 전문가로 보면 됩니다.

**Context (컨텍스트/문맥)** -- 그동안 현재 세션에서 Claude가 사용자와 대화하면서 읽어온 파일, 질문 답변 내역의 누적 등 일종의 '임시 누적 기억'을 뜻합니다.

**MCP Server (MCP 서버)** -- 데이터베이스 연결이나 외부 앱 연동 등 평소 클로드가 못하는 것을 별도로 연장, 장착해서 능력을 확장시키는 플러그인 개념.

## 핸즈온 실습: 당신의 첫 번째 Claude Code 세션 구동

아주 간단하게 무언가를 한 번 만들어봅시다. 중요한 건 완벽함이 아니라 작업 워크플로우를 몸으로 경험해 보는 것입니다.

### 예제 프로젝트: 파이썬 스크립트 작성해 보기

#### 1단계: 프로젝트용 폴더(디렉토리) 하나 만들기

아직 Claude를 켜지 않은 생 터미널 창 상태에서:

```bash
mkdir hello-claude
cd hello-claude
```

`mkdir`는 폴더를 만드는 명령, `cd`는 그 만들어진 폴더 안으로 이동하는 명령입니다.

#### 2단계: Claude Code 실행(Start)

```bash
claude
```

지금 당신은 하나의 '세션(Session)'을 열었습니다.

#### 3단계: 첫 번째 명령 프롬프트 입력해보기

이렇게 입력해 보세요:

```
Create a Python script called hello.py that asks for the user's name and greets them
(사용자 이름을 물어보고 인사하는 hello.py라는 파이썬 스크립트를 만들어줘)
```

Enter 키를 누르세요.

Claude Code가 여러분의 질문을 잘 받았다고 즉각 피드백하고, Write 도구를 사용하여 파일 생성을 끝내고 완료되었다고 보고할 것입니다.

#### 4단계: 정상적으로 파일을 잘 생성했는지 확인하기

자, 이렇게 타이핑해 보세요:

```
Can you show me what's in hello.py?
(hello.py 안에 무슨 내용이 있는지 보여줄래?)
```

Claude Code가 이번엔 Read(읽기) 도구를 사용하여 화면에 파일 내용을 출력해 줄 것입니다.

#### 5단계: 이 녀석이 정말 동작하는 프로그램인지 돌려보기

입력해 보세요:

```
Run the hello.py script
(hello.py 스크립트 실행해)
```

Claude Code가 가장 강력한 조작 도구인 Bash 도구를 써서 `python hello.py`를 실행해 줍니다. 스크립트가 돌아가며 여러분의 이름을 물어보면 자기 이름을 타이핑하면 친절한 인사가 찍힐 겁니다.

#### 6단계: 내용 일부 수정시켜보기

입력하세요:

```
Make the greeting more enthusiastic with exclamation marks
(인사말을 좀 더 열정적으로 느끼게 느낌표를 좀 팍팍 넣어봐)
```

Claude Code가 어떤 코드 영역의 내용을 타겟팅하여 수술하듯이 Edit(편집) 도구를 ഉപയോഗ해 아주 정밀한 변경 패치를 단행하고, 화면에는 구 코드에서 신 코드 버전이 어떻게 수정되었는지 표시해줍니다. 정말 필요한 문자만 똑 떼서 수정합니다.

축하합니다! 방금 첫 파일을 기획부터 생성하고, 정상 동작 확인까지 검증하고, 코드의 요구 변화까지 대화로 이끌어내 완성했습니다!

## 내가 방금 겪은 워크플로우 분석

이게 바로 여러분이 이 도구와 함께 사용할 기본기 동작이자 반복 루프의 정석입니다.

**스크립트 파일 최초 작성:** 사용자가 무엇이 지시사항인지 요점을 던지면 알아듣고, 그에 가장 알맞은 특화 기능인 Write(쓰기) 도구를 선택하여 파일 작성을 훌륭히 수행했습니다.

**해당 파일 정보 열람:** 작업 이후에 Claude가 다시 Read(읽기) 도구를 꺼내 내용을 스캔해서 눈앞의 디스플레이에 노출시켰습니다.

**적용 테스트 실행하기:** Claude가 자기가 만든 코드를 확인하기 위해 Bash(제어 권한 접속기) 도구를 열어서 시스템의 실제 동작 명령을 수행시켜 버렸습니다.

**원하는 부분만 정확 수정:** 전체 코드를 무작정 다시 버리고 새로 짜는 것이 아니라, 마치 전문가 수술의처럼 바꿀 위치만 정확히 도려내서 핀포인트로 살짝 고치고(Edit 도구) 이빨을 맞추는 놀라운 능력을 확인했습니다.

이 "내가 말하고 → AI 도구가 구동되고 → 확인 검증" 하는 구조가 AI 코딩의 모든 것이라 보면 됩니다. 남은 건 그저 이 반복 작업을 어디에 계속 응용할지에 달린 거니까요!

## 자주 묻는 질문(FAQ)

**코딩을 몰라도 (할 줄 아는 언어가 없어도) 되나요?**
물론 가능합니다! 다만 기초적인 컴퓨터 동작의 원리를 파악하거나 알아두면 곁에서 Claude가 일할 때 여러분이 지시하거나 관망하기 훨씬 나으실 겁니다. 배우면서 계속 진도를 빼시면 됩니다.

**이 프로그램 완전 무료인가요?**
Claude Code 앱은 설치해도 돈이 들거나 비용이 청구되지 않은 프리-오픈웨어 앱입니다. 다만 통신을 처리하고 똑똑하게 답안을 내려주는 Claude API 통신 요금과 키는 종량제 결제로 들어갑니다. 최초 사용자는 일정 무료 한도를 받아 실습이 가능합니다.

**내가 명령어를 틀리거나 지시를 병맛같이 내리면 어떡해요?**
말을 바로 정정해서 다시 하거나, 방금 했던 것 다 무효(Undo)로 취소시켜 달라고 그냥 말하면 됩니다. 영구적으로 무언가 날아가거나 실수할 일은 없으니 걱정 마세요.

**어떤 프로그래밍 언어를 다룰 수 있나요?**
여러분 머릿속에 아는 세상의 모든 프로그래밍 언어요. (Java, Python, JS, C, Rust 구별 없이 모두 다)

**Claude Code가 코드를 바보같이 실수하거나 오류내면 어떻게 하죠?**
모든 주도권과 관리 결정은 어쨌든 마우스/키보드 버튼을 쥐고 있는 인간 자신입니다. 최종 사항을 승인 적용하기 전엔 항상 내게 승인 여부를 따져 묻기 때문에 "야, 이거 잘못 짠 것 같아" 하고 다시 빠꾸 수정 지시를 계속 내리세요.

**그냥 이 화면을 통째로 끄고 싶어요.**
`/exit` 을 입력하거나, 키보드의 Ctrl+c 키를 누르세요. 나간다고 코드가 날아가는 일은 없고 다 자동 저장되어 있습니다.

## 중요 팁

1. **말을 좀 더 편안히 해보세요** -- 대화하는 동료처럼 다루세요. 포털 사이트에 검색 키워드 몇 개 넣듯 대충 딱딱거리지 말고 풀 센텐스로 사람 취급해주세요.
2. **구체적으로 말하세요** -- 디테일이 생명입니다! 요구 조건이 타이트하고 정확할수록 무섭고 완벽한 퀄리티가 뽑힙니다.
3. **많이 물어보세요** -- 자신이 무언가 코딩이나 이해가 막히면 혼자 끙끙대지 말고 얘한테 바로 이유와 원리를 물어보세요.
4. **결과를 꼭 확인하세요** -- 그냥 오케이하고 넘기는 버릇을 버리고 코드가 어떻게 생겼는지 눈으로 슬슬 봐야 발전이 있습니다.
5. **실험해 보세요** -- "이런 거 시켜도 될까?" 하고 일단 던져 보세요. 뭐 고장 나거나 시스템 터지는 일 없습니다. 다 복구됩니다.
6. **AI의 답변을 차분히 들으세요** -- Claude는 자기가 하는 일이 무슨 과정인지 잘 브리핑해 줍니다. 이게 가장 확실한 교육 코스 선생님 역할입니다.

## 트러블슈팅 가이드

**"Command not found: claude" 에러 메시지가 떠요.**
설치 명령어 `npm` 사용 시 `-g` 설정을 제대로 지시했는지 확인 후 껐다가 다시 켜세요.

**"API key not found" 에러가 나와요.**
환경변수 등록(`ANTHROPIC_API_KEY`)이나 터미널 컨피그 환경(`claude config set apiKey`)이 무사히 내 컴퓨터 시스템에 세이브되어 적용되는지 확인.

**"Permission denied" 권한 에러.**
맥, 리눅스에서 최고 권한 문제로 생긴 일일 수 있으니 명령어 맨 앞에 `sudo` 를 붙여 임시 실행 강제 적용해 봅니다. (예: `sudo npm install -g claude-code`)

**Claude Code가 너무 반응 속도가 꿀 먹은 벙어리처럼 느려요.**
아주 정상입니다. 복잡한 미션을 지시하면 10~20초 이상 시간이 오래 걸릴 수 있으니 느긋하게 폼 잡고 커피 드세요.

**에러 메시지도 없이 아무 일도 안 일어나요.**
엔터(Enter) 키 안친 것은 아닌지? 아니면 컴퓨터 인터넷 통신 장애(와이파이, 랜선)일 수 있습니다.
