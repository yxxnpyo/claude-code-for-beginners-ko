# 모듈 15: 프로덕션 실전 배포 (Production Deployment)

**목표:** 내 로컬 컴퓨터에서만 돌던 꼬마 앱을, 수천 명이 접속하는 인터넷 세상(Production)에 런칭하기

**예상 소요 시간:** 60-90분

---

## 무엇을 배우게 되나요?

코딩의 진짜 마지막 보스, '배포(Deployment)'입니다. 이 모듈에서는 Vercel이나 Heroku 같은 쉬운 클라우드부터 리눅스 가상 서버(VPS), 그리고 모든 환경을 통일하는 Docker 컨테이너 팩토리얼까지 다룹니다. 클로드 코드를 활용해 배포 셋업 스크립트를 짜고, GitHub Actions를 통한 CI/CD(자동 배포 파이프라인)를 뚫는 법을 배웁니다.

---

## 레슨 1: 배포의 기초 (Fundamentals)

### 배포(Deployment) 란?

내 노트북(Development)에서 내 마음대로 돌아가는 코드를, 지구상 누구나 접속할 수 있는 엄격하고 통제된 서버 컴퓨터(Production)로 이사시키는 과정입니다.

내 개발 환경에선 덤프용 더미 데이터를 쓰지만, 배포 서버엔 진짜 유저 데이터가 들어갑니다. 에러가 터졌을 때 내 화면엔 빨간 스택트레이스가 다 떠도 상관없지만, 배포 서버에서 그런 게 유저 화면에 뜨면 곧바로 해킹의 타깃이 됩니다.

### [필독] 배포 직전 클로드에게 시킬 최종 인스펙션(검열)

```
나 이 프로젝트 드디어 실서버(Production)에 배포할 건데, 올리기 전에 이 항목들 싹 다 스캔하고 문제점 고쳐줘:
1. 보안 위험한 환경변수(.env) 하드코딩 된 곳 없는지 색출!
2. 에러 났을 때 유저 화면에 서버 폴더 경로나 DB 에러 원본 메시지 유출되는(Leak) 곳 없는지!
3. console.log 쓰레기 남발한 거 지우고 진짜 Winston 같은 로깅 프레임워크로 교체
4. 무지성 호출 못 하게 타격 횟수(Rate limiting) 제한 걸어
5. CORS 정책 `*` 로 대충 뚫어논 거 우리 도메인만 되게 막아
6. 서버 살아있는지 헬스체크(Health Check)하는 API 엔드포인트 하나 뚫어
```

## 레슨 2: Vercel에 프론트엔드/풀스택 배포하기

### Vercel이 뭔가요?

Next.js, React 등 프론트엔드 프레임워크와 Serverless 함수들에 완벽하게 최적화된 마법의 호스팅 플랫폼입니다. Next.js를 쓴다면 무조건 여깁니다.

### 클로드에게 세팅시키기

```
이 Next.js 앱 Vercel에 배포할 거임. 세팅 도와줘:
1. `vercel.json` 설정 파일 규격에 맞춰서 생성해
2. 배포할 때 건드려야 할 환경변수 목록만 추려서 줘
```

로컬 터미널에서:

```bash
npm install -g vercel  # CLI 설치
vercel login           # 브라우저 로그인 연동
vercel --prod          # 한 방에 프로덕션 실서버 발사!
```

## 레슨 3: Heroku에 백엔드 배포하기

### Heroku가 뭔가요?

Node.js(Express), Python(Django), Ruby 등 진짜 서버 백엔드를 돌릴 때 가장 스트레스 안 받고 쏘아 올릴 수 있는 유료 플랫폼입니다.

### 클로드에게 세팅시키기

```
내 Express 백엔드 API 서버를 Heroku(헤로쿠)에 배포할 거야:
1. 헤로쿠가 앱 실행법 알아먹을 수 있게 `Procfile` 파일 생성해 줘
2. 내 소스코드 중에 포트(PORT) 번호 하드코딩된 거 있으면 process.env.PORT 따르게 다 고쳐
```

클로드가 짜준 `Procfile`:

```
web: node dist/server.js
```

터미널 발사:

```bash
heroku create 내-앱-이름
heroku config:set NODE_ENV=production
git push heroku main
```

## 레슨 4: 야생의 리눅스 서버 (VPS - Ubuntu) 에 깡으로 배포하기

AWS EC2, Cafe24, DigitalOcean 같은 진짜 빈 깡통 서버를 사서 바닥부터 내 손으로 올리는 겁니다. 엄청난 생고생이지만 쌉니다.

```
이 Node.js 앱을 빈 깡통 Ubuntu 22.04 가상 서버(VPS)에 배포하려고 해. 
다음 터미널 명령어 리스트(스크립트) 싹 다 작성해서 줘:
1. ssh로 서버 들어가서 Node, PM2, Nginx 무설치 깡통 상태에서 설치하는 커맨드들
2. 특정 포트(80, 443, 22)만 열리게 리눅스 방화벽(UFW) 세팅
3. Let's Encrypt로 무료 HTTPS(SSL) 인증서 발급받기
4. Nginx가 방어막 역할하도록 Reverse Proxy 설정 파일(nginx.conf) 템플릿 하나 짜줘
5. PM2 써서 내 앱 죽어도 무한 자동 부활시키게 `ecosystem.config.js` 파일 만들어
```

클로드가 내려준 긴 터미널 지시문들을 긁어서 서버에 복붙하면, 그 거지 같은 인프라 세팅이 5분 만에 끝납니다.

## 레슨 5: Docker (도커) - 싹 다 컨테이너로 묶기

"내 컴퓨터에선 되는데 왜 니 서버에선 안돼?" 이 악몽을 종식하는 기술입니다. 어플리케이션, 리눅스 운영체제, 라이브러리, 데이터베이스까지 통째로 진공 포장해서 어디서든 100% 똑같이 켜지게 만듭니다.

```
이 Node.js 서버 Docker로 말아버릴 거야 세팅해:
1. 용량 젤 작은 alpine 리눅스 기반으로 `Dockerfile` 멀티스테이지 성능 최적화해서 짜!
2. 이상한 로그파일 안 딸려들어가게 `.dockerignore` 빡세게 작성하고
3. 내 Node 앱이랑 PostgreSQL DB 두 개를 한 번에 묶어서 띄울 수 있게 `docker-compose.yml` 파일 설계해 줘.
```

## 레슨 6: CI/CD (GitHub Actions) - 머지(Merge)하면 자동 배포 공격

코드를 푸시할 때마다 내가 직접 서버 접속해서 다운받고 켜는 건 원시인이나 하는 짓입니다.

### "GitHub, 내가 코드 올리면 네가 알아서 서버에 런칭해 놔"

```
GitHub Actions 써서 메인(main) 브랜치에 코드 Push 될 때마다 자동 배포(CI/CD)되는 워크플로우 `.github/workflows/deploy.yml` 파일 작성해!

루틴:
1. 가상컴퓨터(ubuntu-latest) 빈 깡통 켬
2. 의존성(npm ci) 설치
3. 린트(Lint) 검사 및 테스트 코드 전부 실행 (에러 나면 빨간불 띄우고 여기서 멈춤)
4. 테스트 패스하면 빌드(Build)
5. 다크템플러처럼 우리 배포 서버로 SSH 접속해서 방금 빌드한 코드 덮어쓰고 서버 재시작까지!
```

이걸 루트에 올려두면 깃헙에 푸시할 때마다 백그라운드에서 요원들이 테스트를 돌리고 서버를 교체해 줍니다.

## 레슨 7: 절대 망하면 안 되는 '데이터베이스 마이그레이션(Migrations)'

실서버(Production)에 데이터가 10만 개 쌓여있는데, 유저 테이블에 '나이' 컬럼 하나 추가하겠다고 테이블을 지웠다 깔면 어떻게 될까요? 인생 하드코어 모드 시작입니다. 항상 변경된 이력(마이그레이션) 코드를 남겨야 합니다.

```
데이터베이스(DB) 실서버 스키마 업데이트할 거니까 마이그레이션 뼈대 짜줘:
- users 테이블에 'phone_number' 란 컬럼을 추가할 건데,
- 업(Up) 쳤을 땐 추가되는 로직 짜고
- 에러 터져서 비상 대피소(Down) 가동하면 롤백해서 컬럼 다시 없애버리는 롤백(Rollback) 코드 무조건 짜놔!
```

## 모듈 15 요점 체크리스트

모든 튜토리얼을 완주하기 위한 최종 점검:

- [ ] 내 코드가 로컬용인지, 실서버용 배포 준비가 끝났는지 클로드에게 인스펙션(검사) 받을 수 있다.
- [ ] Vercel이나 Heroku 같은 플랫폼의 기본 세팅 파일들을 AI를 통해 1분 만에 뽑아낼 수 있다.
- [ ] 생 리눅스 서버(VPS)에 우분투 세팅부터 Nginx, Let's Encrypt까지 설치하는 터미널 스크립트를 하청 줄 수 있다.
- [ ] 내 프로젝트를 Dockerfile 과 docker-compose 로 진공 포장시킬 수 있다.
- [ ] GitHub Actions 로 테스트와 배포를 자동화(CI/CD)시키는 야망을 품었다.
