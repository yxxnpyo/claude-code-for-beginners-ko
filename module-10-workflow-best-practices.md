# 모듈 10: 개발 워크플로우 베스트 프랙티스 (실무 스킬)

**목표:** 단순한 토이 프로젝트 수준을 넘어, Claude Code로 '프로페셔널'하게 일하는 습관 들이기

**예상 소요 시간:** 35-45분

---

## 무엇을 배우게 되나요?

이 모듈에서는 한 번 쓰고 버릴 스크립트 작성 수준을 넘어, '현업 프로덕션 레벨'의 코드로 올라가는 습관들을 다룹니다. 태스크(할 일) 관리, 코드 리뷰, 주석과 문서화, 에러/보안 처리 전술을 배웁니다. 이것들은 당장 내일, 아니 6개월 뒤 미래의 당신이 과거의 당신 코드를 보고 쌍욕을 박지 않게 만들어 줄 핵심 소양입니다.

## 레슨 1: TodoWrite로 임무(Task) 쪼개서 관리하기

### TodoWrite가 뭔가요?

Claude Code에 내장된 '할 일 관리' 시스템입니다. 뭘 하고 있었는지 추적하고, 한눈에 진척도를 보여줍니다. 미친 듯이 복잡한 작업 중간에 화장실 핑계로 흐름이 끊겼을 때 아주 유용합니다.

---

### TodoWrite는 언제 쓰나요?

수많은 잔업이 엮인 큰 기능(3단계 이상), 기획이 복잡한 작업 등에 씁니다.
간단한 파일 1개 수정 같은 덴 쓰지 마세요. 닭 잡는 데 소 잡는 칼 쓸 필요는 없습니다.

---

### Todo 리스트 생성하기

큰 미션을 던져주며 체계를 잡아달라고 합니다:

```
내 앱에 로그인 인증 시스템 추가해야 해.
이걸 위한 Todo 리스트(체크리스트) 만들고 시작하자:
1. DB에 유저 테이블 세팅
2. 회원가입 엔드포인트 뚫기
3. 로그인 엔드포인트 뚫기
4. JWT 토큰 생성 로직 붙이기
5. 보호해야 하는 API 라우트 막기 (미들웨어)
6. 테스트 모듈 짜기
7. README 문서 업데이트
```

클로드는 이걸 받아 아래와 같은 상태판을 만들어 띄워줍니다:

```
✓ DB에 유저 테이블 세팅 (완료)
→ 회원가입 엔드포인트 뚫기 (진행 중)
○ 로그인 엔드포인트 뚫기
○ JWT 토큰 생성 로직 붙이기
○ (이하 생략...)
```

---

### 진척도 추적하기

Claude Code는 진행 중인 곳에 `in_progress`, 끝난 곳에 `completed` 마킹을 알아서 뗐다 붙였다 합니다.

명령어로 툭툭 건드려 보세요:

```
지금 내 진행 중인 투두(Todo) 리스트 현황판 좀 화면에 띄워 줘
나 지금 뭐까지 했고 뭐 남았지?
3번 태스크 완료 처리 해 줘
```

## 레슨 2: 코드 리뷰 (Code Review) 기술

### 커밋(Commit) 전 스스로 매 맞기

커밋하기 전에 항상 리뷰를 시키세요. 영구 박제되기 전에 바보 같은 짓을 막는 제일 쉬운 길입니다.

```
내가 깃 커밋 날리기 직전인데, 아래 항목들 다 체크해서 보고해 줘:
1. 변경된 모든 파일 리스트업
2. 파일별로 뭘 건드렸는지
3. 혹시 터질 만한 잠재적 에러가 있는지
4. 더 세련되게 고칠 개선안은 없는지
```

---

### 가혹한 셀프 감리 (Self-Review Checklist)

클로드한테 전문가 뺨치는 체크리스트를 던지며 물어뜯으라고 지시하세요:

```
방금 짠 내 코드들 싹 리뷰하고 이 목록들 다 검사해:
- 보안 취약점 (뚫릴 만한 곳)
- 퍼포먼스(속도) 저하 원인
- 에러 핸들링 빼먹고 대충 넘긴 곳
- 쓸데없이 중복된 겹치는 코드
- 아무나 봐도 모를 쓰레기 같은 변수명
- 안 짠 방어막(테스트) 코드
```

---

### 남의 코드 리뷰하기 (Pull Request Review)

남이 짠 PR 코드를 읽어봐야 할 때 클로드 대리님을 부릅니다:

```
나 지금 팀원이 올린 이 PR(풀 리퀘스트) 코드 리뷰해야 돼.
좀 도와줘:
1. 이 인간이 이 코드의 목적을 파악해 줘
2. 짱박혀 있는 버그 없는지 다 찾아
3. 뭔가 좀 더 깔끔하게 짤 수 있는 곳 지적해 줘
```

## 레슨 3: 문서화 (Documentation)

### 코드 주석 (Code Comments) 달기

명심하세요: **좋은 주석은 "무엇(What)"이 아니라 "왜(Why)"를 설명합니다.**

**존멋 주석 (O):**

```javascript
// 외부 API 장애가 났을 때 우리 서버마저 뻗는 걸 막기 위해
// 지수 백오프 기반으로 호출 재시도(Retry) 시간 텀을 의도적으로 두었습니다.
const delay = Math.pow(2, retryCount) * 1000;
```

**병신 주석 (X):**

```javascript
// retryCount 값을 제곱해서 1000을 곱함
const delay = Math.pow(2, retryCount) * 1000;
```

코드 보고 아는 사실을 한글로 번역만 하는 주석은 다 치워버리세요.

### 클로드에게 주석 하청(?) 맡기기

```
userService.js 파일에 있는 모든 함수 위에 JSDoc 스타일 주석 다 박아 놔:
- 함수 목적 요약
- 파라미터 타입이랑 이름 설명
- 리턴 반환값 타입
- 언제 어떤 에러 튕겨내는지 스펙
```

---

### README.md 대필 시키기

README 문서는 당신 프로젝트의 얼굴(현관문)입니다. 개떡같이 쓰면 아무도 안 봅니다.

```
이 프로젝트 루트 폴더에 README.md 파일 하나 기깔나게 써 줘.
들어갈 내용:
- 멋진 프로젝트 소개말
- 핵심 기능 요약 리스트
- 돌리기 전에 필요한 준비물 (Prerequisites)
- npm install 및 실행 가이드
- 환경 변수(.env) 설정법 설명
```

## 레슨 4: 에러 짬처리 금지 (Error Handling)

에러 핸들링 대충 퉁치고 넘기는 건 나중에 새벽 2시에 회사 서버 터져서 당신 폰이 울리게 만드는 원흉입니다.

```
userService.js DB 쿼리 부분에 에러 핸들링 빡세게 추가해 줘:
- 모든 비동기(Async) 쿼리에 Try/Catch 방어벽 다 둘러
- 터진 이유별로 명확한 메시지를 토하게 해 줘
- 유저 화면에는 "오류가 낫습니다" 라고만 보여주고, 실제 DB 스키마 에러 같은 민감한 코드는 절대 밖으로 안 새어 나가게 막아!!
```

### 멋진 Custom Error 정의하기

```
앱에서 돌려쓸 커스텀 에러 클래스들 한 땀 한 땀 다 만들어:
- ValidationError (400 상태코드)
- AuthenticationError (401)
- AuthorizationError (403)
- NotFoundError (404)
- DatabaseError (500)
```

## 레슨 5: 범인 찾기용 로깅 (Logging) 전술

### 무엇을 남기고, 무엇을 지울 것인가

**반드시 로그 쾅쾅 박아야 하는 것:**

- 누가 로그인했다, 데이터 뭐 지웠다 같은 민감 조작
- 에러 터졌을 때의 스택 트레이스 흔적
- 외부 API나 결제 모듈 호출 결과

**절대 로그 텍스트로 남기면 안 되는 미친 짓:**

- 유저 비밀번호 (진짜 쇠고랑 찹니다)
- 주민번호 같은 개인 정보

```
앱 전체에 쓸 수 있는 Logger 유틸리티 하나 파줘:
- error, warn, info, debug 레벨 다 쪼개고
- 각 로그엔 시간(Timestamp), 접속자 IP 주소 다 찍히게 해
```

## 레슨 6: 폴더 정리 정돈 (Code Organization)

### 끼리끼리 모아놓기 (Feature 기준 구조)

models, controllers, routes 폴더에 수천 개 파일 쑤셔 박지 말고, 차라리 도메인(유저, 게시판) 별로 폴더를 자르세요.

```
이 앱 구식 폴더 구조 좀 최신 트렌드로 다 뜯어고쳐 줘:
기존 방식 (종류별로 묶음)
src/routes/, src/controllers/, src/models/

바꿀 방식 (기능 테마별로 묶음)
src/users/ 안에 (모델,컨트롤러,라우터,테스트) 다 몰빵
src/tasks/ 안에 (모델,컨트롤러,라우터,테스트) 다 몰빵
```

### 중복 소스코드 청소 (DRY 원칙 - Don't Repeat Yourself)

똑같은 코드를 세 군데 복붙했다고요? 한 군데 버그 나면 세 군데 다 찾아가서 고쳐야 합니다.

```
이 프로젝트 다 스캔해서 완전히 중복되거나 복붙한 냄새나는 로직 싹 다 잡아내서 중복 제거 리팩토링 싹 다 해.
따로 Utils 모듈로 빼서 재사용하게 묶어!!
```

## 레슨 7: 절대 뚫리면 안 되는 보안 방벽 (Security)

이건 진짜 타협 제로 구역입니다.

```
이 앱에 무조건 깔려야 하는 보안 방호벽(Security) 싹 다 풀가동 시켜:
- 유저한테 입력받는 모든 칸에 Validation 씌우기
- SQL 인젝션 공격 못 들어오게 쿼리문 파라미터화(parameterize) 처리
- 암호 변경 실패 n번 연속 나면 계정 잠가버리기(Lockout)
- Helmet.js 모듈 써서 HTTP 보안 헤더(headers) 방패 둘러치기
```

**그리고 가장 많이 당하는 초보자 실수:** API 키코드(.env)를 생으로 깃헙에 올리는 짓.

```
내 코드에 혹시 하드코딩(코드 안에 날것으로 박혀있는) 된 비밀키나 패스워드 없는지 싹 스캔해서 검사해! 있으면 다 뽑아서 .env 변수로 우회시켜!!
```

## 레슨 8: 퍼포먼스 튜닝 (Performance)

```
데이터베이스 조회 속도 튜닝 좀 해줘:
1. 검색 자주 일어나는 컬럼에 Indexing(색인) 쿼리 적용
2. 미련하게 100개 다 돌리는 N+1 쿼리 문제 찾아내서 JOIN 문으로 병합시켜
3. 한방에 데이터 수만 개 안 불러오게 Paging(페이지 나누기) 로직 무조건 강제로 달아 놔.
```

## 모듈 10 요점 체크리스트

과정을 끝내기 전 셀프 점검:

- [ ] TodoWrite 기능으로 개발 미션을 작게 썰어 트래킹할 수 있다.
- [ ] 내 코드를 커밋 직전에 AI에게 리뷰(감리)받아 수정하는 버릇이 들었다.
- [ ] JSDoc과 README.md를 고퀄리티로 자동 대필시킬 줄 안다.
- [ ] 에러를 무시하거나 대충 뱉지 않고 깐깐한 커스텀 에러로 핸들링한다.
- [ ] 로그(Logging)를 남길 것과 안 남길 것을 구별할 수 있다.
- [ ] 보안 취약점과 퍼포먼스 최적화를 클로드에게 강제로 지시할 줄 안다.
