# 모듈 3: Claude Code의 도구(Tools) 이해하기

**목표:** Claude Code가 사용할 수 있는 여러 다채로운 도구들을 마스터하기

**예상 소요 시간:** 40-50분

---

## 무엇을 배우게 되나요?

이 모듈에서는 Claude Code가 백그라운드에서 사용하는 여러 도구들에 대해 알아봅니다. 각 도구의 역할이 무엇인지, Claude가 언제 어떤 도구를 꺼내 사용하는지, 실제 업무를 처리하기 위해 어떻게 이들을 연계하는지 배웁니다. 또한 원할 때 Claude를 내가 직접 원하는 도구 쪽으로 조종(Steer)하는 방법도 알게 됩니다.

---

## 레슨 1: 도구(Tools)란 무엇인가요?

### 도구 이해하기

도구란 Claude Code가 당신의 시스템 환경과 코드베이스에 접근하여 상호 작용하기 위해 탑재된 특정 기능들입니다. 각 도구는 다음 한 가지 작업을 매우 뛰어나게 수행합니다:

- **읽기(Reading)** -- 문서나 파일을 쳐다보고 파악합니다
- **쓰기(Writing)** -- 완전히 새로운 문서를 작성해 냅니다
- **수정하기(Editing)** -- 기존 문서의 살을 도려내거나 붙여 변경합니다
- **명령 실행(Running commands)** -- 터미널 창을 조작해 액션을 수행합니다

이게 거의 모든 것의 전부입니다. Claude Code는 당신의 지시를 분석하여 가장 알맞은 단일 도구(또는 도구들의 연속 조합)를 자동으로 골라서 사용합니다.

### Claude Code의 도구 사용 과정

당신이 요청을 던지면, Claude Code는 이 심플한 과정을 밟습니다:

1. 사용자의 요청을 분석하고 이해한다
2. 어떤 도구들을 쓸지 고른다
3. 도구를 실제로 구동한다 -- 당신의 화면에 이것이 실시간으로 노출됩니다
4. 그 도구가 뱉어낸 결과를 당신에게 화면으로 보여준다
5. 방금 자신이 무슨 작업을 완료했는지 요약 브리핑을 한다

실제로는 이렇게 돌아갑니다:

```
당신: "Hello World 출력하는 파이썬 파일 생성해"

Claude Code 내부 생각:
- 새 파일이 필요하구나 → Write 도구 당첨
- 어차피 지금 파일이 없으니까 → Write만 쓰면 되겠군
- 파이썬 코드로 출력을 작성해야지

Claude Code 실제 행동:
- Write 도구를 써서 hello.py 생성 완료
- 완료되었다고 사용자에게 보고
```

## 레슨 2: 파일 컨트롤 도구들

### 1) The Read (읽기) Tool

**목적:** 파일 내용을 읽고 스캔하기 위해

**언제 사용되나요:**

- 당신이 "파일 내용 좀 보여줘" 할 때
- 코드를 수정하기 전, 기존 코드를 이해하기 위해
- 무언가 코드를 변경한 뒤 변경이 예쁘게 잘 들어갔나 자기 점검을 할 때

**사용 예시 화면:**

```
Reading app.js...

[파일 내용이 라인 번호와 함께 출력됩니다]
```

Read 도구는 100% 비파괴적(non-destructive)입니다 -- 그 어떤 파일도 건드리거나 손상시키지 않는 가장 기본적이고 안전한 눈(Eye)의 역할을 합니다.

---

### 2) The Write (쓰기) Tool

**목적:** 전에 없던 새로운 파일들을 빚어내기 위해

**언제 사용되나요:**

- 아예 존재하지 않는 새 파일들 만들 때
- 기초 환경 설정 파일을 깔 때
- 문서(Documentation)나 README를 적어낼 때
- 테스트용 코드를 모아둔 파일 세트를 만들 때

**알아둘 점:** Write 도구는 '새로운' 파일을 태어나게 하는 역할입니다. 만약 이미 동일한 이름의 파일이 존재한다면 덮어쓰지 않고 Edit(수정) 도구를 꺼내듭니다.

---

### 3) The Edit (수정) Tool

**목적:** 이미 존재하는 기존 파일에서 완벽히 특정 부위만 정밀 교체/수정하기 위해

**언제 사용되나요:**

- 만들어져 있는 코드를 손볼 때
- 에러 버그 고치기
- 무언가 기존 함수에 살을 붙여 새 기능 만들 때
- 코드 다이어트(리팩토링) 할 때

**사용 예시 화면:**

```
Editing server.js...

Old(이전):
const PORT = 3000;

New(새것):
const PORT = 8080;
```

이건 정말 엄청나게 중요합니다 -- Edit 도구는 파일 전체를 무작위로 덮는 짓을 절대 하지 않습니다. 옛날 버전과 새 버전의 차이(diff)를 핀포인트로 정확히 찔러서 고칩니다. 여러분은 이 Edit 도구가 출력한 흔적을 눈으로 확인하는 습관을 들여야 합니다.

---

### 4) The Glob (글롭) Tool

**목적:** 이름이나 확장자 등 특정 패턴을 갖춘 파일 뭉치를 통째로 솎아 찾기 위해

**언제 사용되나요:**

- 특정 유형(예: .js) 파일만 싹 다 긁어모아 보고 싶을 때
- 내 프로젝트 폴더 한구석에 박힌 특정 파일을 찾아야 할 때
- 프로젝트 트리(구조) 모양을 파악할 때

**검색 예시:**

```
*.js → 자바스크립트 파일 전부 다
src/**/*.py → src 폴더와 그 아래 모든 하위 폴더에 있는 파이썬 파일 싹 다
**/*test* → 파일명 중간에 "test" 들어간 거 전부 다
```

---

### 5) The Grep (그렙) Tool

**목적:** 파일의 껍데기가 아니라 속(내용물)에 쓰인 코드나 텍스트 글자를 통째로 검색하기 위해

**언제 사용되나요:**

- "이 변수 도대체 앱 어디어디에 쓰였어?" 찾을 때
- 특정 함수의 근본 뼈대(정의)를 찾고 싶을 때
- 코드 곳곳에 박힌 TODO(미루기) 주석 메모들만 싹 모아 볼 때

**검색 예시:**

```
function login → login이라는 함수 내용 찾아봐
TODO → TODO 주석 전부 다 가져와
class.*User → User란 단어가 포함된 모든 클래스 덩어리 찾아봐
```

Grep 도구는 정규표현식(Regex)을 완벽히 다룰 줄 압니다. 당신의 상상 이상으로 강력하고 무시무시한 코드 내용 추적 능력을 발휘합니다.

---

## 레슨 3: 명령어 실행 도구들

### 1) The Bash (배시) Tool

**목적:** 터미널/쉘(Shell) 명령어 실행 권한

**언제 사용되나요:**

- 구동에 필요한 패키지 다운로드/설치 시 (`npm install`)
- 여러분의 서버 앱 전원을 켤 때
- Git 커밋이나 브랜치 넘길 때
- 테스트 코드를 실제 구동할 때
- 그 외 터미널 창에서 치는 모든 상황

**사용 예시 화면:**

```
Running: npm install

[설치 진행 중...]

✓ Dependencies installed successfully
```

Claude Code는 그 어떤 Bash 명령어라도 일단 당신의 컴퓨터를 조작해 실행하기 전에 "이런 걸 깔고/돌리고/실행해도 되겠습니까?" 라고 당신에게 설명하고 동의를 구합니다. 멋대로 당신의 시스템을 휘젓지 않습니다.

---

### 2) 백그라운드 프로세스 (Background Processes)

**목적:** 실행 후 한참 동안 켜놔야 하는 모터나 엔진 같은 명령어 켜두기

**이럴 때 씁니다:**

- 항상 켜둬야 하는 개발용 로컬 서버
- 파일 저장할 때마다 도는 테스트 감시기(Watch mode)
- 데이터베이스 구동기

**관리 방법 예시:**

```
서버를 저 뒤쪽 백그라운드 환경 구동으로 돌려두고 켜 놔
뒤에서 돌고 있는 백그라운드 서버 좀 꺼줄래
지금 내 컴퓨터 뒤에서 돌고 있는 프로세스 목록들 좀 읊어봐
```

## 레슨 4: AI 특화 도구들

### 1) The Task Tool (전문화된 에이전트 파견)

**목적:** 사람 손이 여러 번 가는 고난도의 복잡한 절차를 스스로 판단하고 완수하기 구동

이런 짓에 특화되어 있습니다:

- 거대하고 방대한 낯선 코드 전체 파악하기
- 새 기능 구현을 위한 밑바탕 기획 및 설계 플랜 짜기
- 버그 연쇄 폭발을 막기 위해 앱 전체 모니터링하기

이 아래 세 가지 특별 부서장(에이전트)이 있습니다:

#### 1. Explore (탐험) 에이전트

```
목적: 코드베이스 이해 및 스캔 특화
이럴때 지시하세요:
- "이 코드베이스 어떻게 굴러가는 놈이야?"
- "이 앱에서 돌아가는 모든 API 구멍(엔드포인트) 싹 찾아내봐"
```

#### 2. Plan (설계/기획) 에이전트

```
목적: 구현 전략 수립 및 방향성 점검
이럴때 지시하세요:
- "이번에 유저 로그인 기능 추가할 건데 설계도 좀 기획해 봐"
- "이 기능에 걸맞은 데이터베이스 테이블 구조 짜서 보고해"
```

#### 3. General-Purpose (일반/다목적) 에이전트

```
목적: 복합적이고 까다로운 장기 퀘스트 전담
이럴때 지시하세요:
- "내 앱에 있는 보안 취약점 싹 다 추적해서 하나하나 다 막아 놔"
- "이 코드베이스 전반에 걸쳐 속도 최적화 작업 싹 돌려줘"
```

에이전트 기능의 가장 소름 돋는 장점은 그들이 "자율성(Autonomous)"을 띈다는 것입니다. 알아서 폴더를 뒤지고 파일을 열고 고치고 실패하고 또 고치면서 자기가 판단하며 끈질기게 임무를 완수하고 당신에게 최종 보고를 합니다.

---

### 기타 유용한 도구들: Search, Fetch, LSP

- **WebSearch 도구:** 인터넷 세계 구글링을 해서 공식 문서, 에러 해결법 등을 최신으로 찾습니다.
- **WebFetch 도구:** 여러분이 툭 던진 인터넷 주소 링크(Github나 공식 Docs 등)를 타고 들어가 그 글 내용을 쑥 빼 읽고 파악합니다.
- **LSP 도구:** 완벽한 코드 추적 시스템. 어떤 함수를 누가 제일 처음 만들었고, 어떤 놈들이 끌어다 쓰고 있는지 족보를 싹 다 추적합니다.

## 레슨 5: 각 도구가 연계되어 춤추는 방식

### 실제 사례: 새로운 기능 추가하기

**나의 요청:** "기존 데이터베이스에서 유저 목록 전체를 던져주는 새 API 엔드포인트 `/users` 하나 뚫어줘"

자, 뒤에서 클로드는 이 짓을 벌입니다:

1. **[Glob 도구 발동]** 기존 라우트(연결) 파일들이 어딨는지 폴더 뒤져 찾아냄.
2. **[Read 도구 발동]** 찾은 파일을 열람하고 기존 룰 파악.
3. **[Grep 도구 발동]** 다른 API들은 어떻게 짜여졌는지 예제를 찾아 복사.
4. **[Edit 도구 발동]** 찾아낸 패턴과 룰셋을 똑같이 따라 해서 `/users` 코드를 기가 막히게 변형하여 이질감 없이 삽입(수정)함.
5. **[Bash 도구 발동]** 앱 서버를 내렸다 다시 켜서 지금 짠 코드가 잘 터지는지(호출 테스트) 자기가 돌려보고 완벽함 확인.

## 레슨 6: 당신이 도구를 강제로 지정해 버리는 방법

Claude가 알아서 도구를 고르는 게 99%지만, 여러분이 개입(Steer) 할 권리도 있습니다.

비슷해 보이지만 다른 뉘앙스들:

**느슨한 의존** (Claude에게 자유도를 줌):

```
app.js에 난 버그 좀 찾아 고쳐봐.
```

**과정과 규칙을 지시** (선 작업, 후 작업 지시):

```
데이터베이스 연결하는 곳들 싹 다 찾은 다음에, 나한테 그 Config 파일 내용물 다 까서 먼저 허락 맡아(보여줘)
```

**특정 도구를 대놓고 쓰라 지정:**

```
grep 도구 써서 TODO 라고 쓴 곳 싹 잡아내
test 파일 싹 다 읽어버려(Read)
Linter(규칙 프로그램) 돌려봐(Bash)
```

언제 지시하는 게 좋을까요? 잘 모르겠으면 클로드에게 운전대를 맡기고, 내가 원하는 과정(예: 고치지 말고 일단 찾은 다음 내 허락을 받아라 등)이 명확할 때만 디테일하게 명령 권한을 지정하세요.
