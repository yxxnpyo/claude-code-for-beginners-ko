## 모듈 1 챌린지 - (클로드 코드 입문)

**과제:** Claude Code를 성공적으로 설치하고 첫 번째 파이썬 프로그램을 창조하라.

**해결 접근법:**

1. **설치하기:**

   ```bash
   # 터미널 준비 (Node.js 18 이상 필수)
   npm install -g claude-code

   # API 키 등록 (당신의 앤스로픽 킷값)
   export ANTHROPIC_API_KEY="your-key-here"

   # 제대로 깔렸는지 생사 확인
   claude --version
   ```

2. **첫 프로그램 명령하기:**

   ```bash
   mkdir my-first-project
   cd my-first-project
   claude
   ```

   채팅창이 뜨면 이렇게 던지세요:

   ```
   greeting.py 라는 파이썬 스크립트 하나 파일로 생성해.
   조건:
   - 실행하면 유저한테 이름이 뭔지 프롬프트로 물어볼 것
   - 가장 좋아하는 색상이 뭔지 물어볼 것
   - 대답을 조합해서 시적인(또는 웃긴) 환영 문구를 그 색상 이름과 맞물려서 출력해 줄 것
   ```

3. **테스트 (실행결과 확인):**

   ```
   방금 만든 greeting.py 스크립트 터미널에서 실행해 봐
   ```

## 모듈 2 챌린지

### 미션 1: 초심자 - 노트 기록용 API 백엔드 깎기

**과제:** Express.js를 써서 아주 간단한 기초 메모장 REST API 서버 구축

**해결 프롬프트 접근:**

```
Express.js 써서 초간단 노트(메모장) REST API 뼈대 짤 건데 요구사항 지켜:
- DB 연결 굳이 하지 말고 그냥 메모리 Array(배열) 변수에 저장해
- 메모 1개의 데이터 구조는: id, title, content, createdAt(생성일자)
- API 주소 구멍 5개 뚫기: GET /notes, GET /notes/:id, POST /notes, PUT /notes/:id, DELETE /notes/:id
- POST 들어올 때 title 이랑 content 항목 안 적었으면 바로 컷(Validation 방어) 시키고 400 뱉어
- 에러 처리랑 HTTP 상태 코드 정확하게 지켜
```

**테스트 명령:**

```bash
# 구동 후 터미널에 쏴보기 (새 메모 저장)
curl -X POST http://localhost:3000/notes \
  -H "Content-Type: application/json" \
  -d '{"title":"비밀계획","content":"내일 모레 퇴사한다"}'
```

### 미션 2: 중급자 - Next.js 풀스택 연락처 폼 만들기

**과제:** Next.js 로 이메일 찌라시 넣는 컨택트 폼 화면 + API 만들기

**해결 접근법:**

**스텝 1: 백지 도화지 깔기**

```
최신 Next.js 프로젝트 새로 터미널에 깔아줘 (TypeScript, Tailwind CSS 무조건 포함시켜서 폴더명은 contact-form-app)
```

**스텝 2: 프론트 UI 그리기**

```
/contact 페이지를 이쁘게 만들어 봐:
- 이름 칸 (필수, 최소 2글자)
- 이메일 칸 (필수, 이메일 정규식 폼)
- 남길 메시지 칸 (필수, 최소 10글자)
- 제출 전송 버튼
- 제출되는 동안 뱅글뱅글(Loading/Spinner) 도는 상태
- 성공/실패했을 때 토스트 알림 메시지 띄우기
- Tailwind로 개간지나게 디자인할 것
```

**스텝 3: 받는 백엔드 구멍 뚫기**

```
/api/contact 서버 API 라우터 뚫어:
- 프론트에서 데이터 넘어오면 유효성 검사 (빈칸 없는지)
- 뚫렸으면 400 에러, 진짜면 200 성공 뱉기
- (일단 이메일 쏘는 기믹은 나중에 하니까 그냥 통신 성공 여부만 확인하게 짜)
```

**스텝 4: 둘이 결합**

```
아까 만든 프론트 폼이랑 백엔드 API /api/contact 통신되게 연결시켜(fetch 든 axios 사용). 성공하면 폼 입력칸 내용 싹 비우는 초기화 로직 잊지 말고!
```

### 미션 3: 고급자 - 진짜 오픈소스 프로젝트 숟가락 얹기 (풀 리퀘스트)

**과제:** 깃허브 구경꾼에서 벗어나 실제 오픈소스 프로젝트에 커밋 기여(Contribution) 해보기

**해결 접근법:**

*팁: 가장 쉬운 타겟: 'first-contributions', 'awesome-*' 위키 리스트, 문서 오타(Docs) 잡기 프로젝트들.*

**스텝 1: 타겟 찌르기**

```
저기 https://github.com/firstcontributions/first-contributions 저장소 URL 통째로 읽어보고, 이 프로젝트 구조랑 기여(Contribution)하는 방식 3줄 요약해 줘.
```

**스텝 2: 파밍 세팅**

```
나 이거 기여할 거니까 터미널로 다 도와줘:
1. 내 깃헙 계정으로 이 저장소 Fork 떠줘
2. 내 로컬 컴퓨터로 Fork한 거 Clone 땡겨오고
3. 원본(Upstream) 저장소 주소도 동기화하게 리모트(Remote) 달아놔 줘
4. 여기 기여 가이드 문서 기반으로 새로운 Branch(작업실) 하나 파고 넘어가!
```

**스텝 3: 수정 (내 이름 끼워 넣기)**

```
Contributors.md 파일에 내 이름 석 자 추가하고 싶어:
1. 기존 써있는 형식(포맷) 그대로 맞추고
2. 알파벳 순서대로 내 이름 들어갈 라인에 껴 넣어 줘
3. 이 짓이 가이드 스펙에 맞는지 마지막으로 한 번 확인 검수해!
```

**스텝 4: PR 서류 생성 및 결재 올리기**

```
이 오픈소스 프로젝트애서 주로 쓰는 깃 커밋(Commit) 컨벤션 메시지 참고해서 내 변경사항 커밋하고 통째로 내 Fork 저장소로 Push 쏴 줘!

그리고 원본 본진 깃허브에 날릴 풀 리퀘스트(PR) 제목이랑 본문 양식 기깔나게 작문해 봐.
```

## 튜토리얼 과제 격파를 위한 팁 모음 (Tips for Challenges)

1. **지시를 잘게 토막 내라:** "인스타그램 클론 만들어줘" 하면 망합니다. "1. 로그인 화면 뼈대 짜 -> 2. 그 다음 댓글 DB 세팅해 -> 3. 댓글 입력 폼 API 연결해..." 처럼 잘라 먹이세요.
2. **"왜 터졌는지" 물어봐라:** 에러 나서 멈췄으면 당신이 고치려 들지 말고 에러를 그대로 긁어서 `어디 찍어 먹고 고장 난 건지 분석해` 라고 던지세요.
3. **공부 모드를 켜라:** 얘가 짠 코드가 내 이해를 넘어섰다면 그냥 복붙하지 말고 `방금 네가 갈아엎은 로직, 무슨 디자인 패턴 쓴 건지 한 줄 한 줄 주석 달아서 한국어로 해설해` 라고 겁박하세요.

## 최악의 헛발질(대참사) 삽질 유형 베스트 (Common Pitfalls)

- **삽질 1: 두루뭉술한 신선놀음** - "API 하나 만들어줘" (어쩌라고. 포스트그레스 쓴다고? 프레임워크는?)
- **삽질 2: 한 입에 다 욱여넣기** - "프론트, 백, 디비, 배포 싹 다 만들어줘." (중간에 무조건 꼬입니다. 모듈별 스텝바이스텝이 정답)
- **삽질 3: 확인 사살 안 하는 놈** - 10개 파일 다 고쳐놓고 테스트 안 돌려놓고 "고쳤습니다" 하는 클로드를 믿으면 바보입니다.
- **삽질 4: 맹신주의** - 클로드가 모르는 프레임워크를 아는 척하며 함수를 환각(지어냄)할 때, `검색(WebSearch) 안하고 어디서 약을 팔아. 당장 공식홈페이지 문서 읽고 와서 다시 짜!!` 라고 다그치십시오.
