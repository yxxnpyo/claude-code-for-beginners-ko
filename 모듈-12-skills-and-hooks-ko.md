# 모듈 12: 스킬과 훅(Skills & Hooks) - 내 맘대로 개조하기

**목표:** 단순 채팅을 넘어, 나만의 단축키(명령어)를 만들고 특정 상황에 AI가 자동으로 움직이게 하는 자동화 세팅법 배우기

**예상 소요 시간:** 45-60분

---

## 무엇을 배우게 되나요?

이 모듈에서는 Claude Code를 당신만의 완벽한 맞춤형 AI 비서로 개조합니다. 프로젝트 전체의 기억을 관리하는 `CLAUDE.md`, 내가 지시하지 않아도 특정 폴더에선 자동으로 규칙을 지키게 하는 `Rules`, 나만의 고유한 명령어를 만드는 `Skills(`/슬래시 명령어`)`, 그리고 코드 수정 직후 자동으로 포맷터(Prettier)나 린터를 돌리게 만드는 `Hooks` 등 끝판왕 커스터마이징을 다룹니다.

---

## 레슨 1: CLAUDE.md & 인공지능의 뇌(Memory) 조작하기

### 세상에서 제일 중요한 파일

프로젝트 루트 폴더에 놓여있는 `CLAUDE.md`는 Claude의 **영구적인 두뇌(기억장치)**입니다. 이게 없으면 클로드는 매번 채팅창을 새로 열 때마다 치매 걸린 사람처럼 "이 앱은 무슨 언어인가요? 코딩 컨벤션은 뭔가요?" 라고 반복해서 묻게 됩니다.

가장 빠르게 기초 두뇌를 생성하려면 터미널에 칩니다:

```bash
/init
```

### CLAUDE.md 에 들어가야 할 진짜 내용 (150줄 제한 권장)

길면 읽느라 시간만 낭비합니다. 핵심만 쓰세요.

```markdown
# 프로젝트: 내 멋진 앱

## 아키텍처 큰그림
- 프론트는 /src/app 에 있는 Next.js 씀
- 백엔드 API는 /src/api 에 Express.js 씀
- 데이터베이스는 PostgreSQL이고 ORM은 Prisma임

## 내가 자주 쓰는 터미널 명령어
- 빌드: `npm run build`
- 테스트: `npm test`
- 서버 켜기: `npm run dev`

## 코딩 스탠다드 (절대 지켜야 할 법)
- TypeScript 무조건 엄격 모드, `any` 타입 금지
- Default export 절대 쓰지 말고 Named export 쓸 것
- 에러 터지면 무조건 커스텀 AppError 클래스로 던질 것
- 테스트 코드는 기존 소스코드랑 같은 레벨 폴더에 (*.test.ts) 둘 것

## 깃 커밋(Commit) 작성 규칙
- feat: 새 기능
- fix: 버그 수정
- refactor: 리팩토링

## 앱 생사결 필수 파일들
- /src/api/middleware/auth.ts - 보안(로그인) 미들웨어
- /src/lib/db.ts - DB 연결 로직
```

자잘한 설명이 너무 길어지면 파일 하나에 다 쓰지 말고 다른 README파일을 참조하라고 `@import` 를 거세요.

```markdown
상세한 API 설명서는 @docs/api-reference.md 파일에 있으니 거기서 읽을 것.
```

## 레슨 2: Rules (모듈화된 세부 규칙)

### 특정 폴더 전용 법률 제정하기

프로젝트가 엄청 크면, 프론트 폴더랑 백엔드 폴더의 코딩 규칙이 다를 수 있습니다. 이럴 땐 `.claude/rules/` 폴더 안에 마크다운을 찢어서 관리합니다.

```
.claude/rules/
  ├── code-style.md
  ├── testing.md
  └── security.md
```

**특정 경로(Path) 한정 룰 지정:**
파일 맨 위에 YAML(프론트매터) 설정으로 발동 조건을 겁니다.

```markdown
---
paths:
  - "src/api/**/*.ts"  # 이 경로의 파일을 건드릴 때만 발동!
---

# API 개발 무조건 지킬 룰

- 모든 엔드포인트에 1차 유효성 검사 미들웨어 끼워 넣을 것
- 에러 응답은 무조건 표준 에러 포맷 쓸 것
```

## 레슨 3: 세팅 (Settings) 계급도

"클로드 세팅을 바꿨는데 왜 안 먹히지?" 무조건 권한(우선순위) 계급 때문에 씹히는 겁니다.

| 우선순위 | 계급 | 위치 | 범위 |
|----------|------|------|------|
| 1짱 | 터미널 옵션 | 명령어 칠 때 `--model` 옵션 등 | 한 번 쓰고 끝 |
| 2짱 | 로컬 개인설정 | `.claude/settings.local.json` | 이 프로젝트 나 혼자 (.gitignore 됨) |
| 3짱 | 플젝 공식설정 | `.claude/settings.json` | 이 프로젝트 전체 팀원 공유 |
| 4짱 | 내 PC 로컬 | `~/.claude/settings.local.json` | 내 컴퓨터 (나 혼자) |
| 5짱 (막내)| 내 PC 글로벌 | `~/.claude/settings.json` | 내 컴퓨터 전체 |

### 미친 짓 방지용 보안 세팅 (Permissions)

`.claude/settings.json`에 클로드가 절대 못 건드리게 막거나 통과시킬 명령을 지정할 수 있습니다:

```json
{
  "permissions": {
    "allow": [
      "Bash(npm run *)",
      "Edit(/src/**)"
    ],
    "deny": [
      "Bash(git push --force *)",   // 강제 푸시 절대 금지!!
      "Read(.env)"                  // 환경변수 비밀키 파일 절대 읽지 마!!
    ]
  }
}
```

## 레슨 4: 스킬 (Skills) - 나만의 '슬래시 명령어' 연성하기

### 스킬의 정체

치트키처럼 `/리뷰해줘`, `/커밋해줘` 같은 나만의 전용 명령어를 만드는 기능입니다. 복잡한 파이썬 스크립트? 필요 없습니다. 그냥 마크다운(`.md`) 파일 하나 쓰면 끝입니다.

**위치:** `.claude/skills/review-code/SKILL.md` 처럼 생성합니다.

### 나만의 코드 리뷰 스킬(/review-code) 뚝딱 만들기

```yaml
---
name: review-code
description: 내 코드를 쓱 스캔하고 훈수를 둔다.
---

코드 리뷰어 봇으로 변신해서 다음을 수행해:

1. `git diff`를 쳐서 내가 방금 짠 코드를 다 스캔해.
2. 뚫릴 만한 보안 이슈나, 너무 느리게 짠 바보 같은 루프문을 찾아내.
3. 3단계로 쪼개서 리포트해 줘:
   - **치명적:** (merge 하기 전에 당장 고쳐야 할 것)
   - **경고:** (고치면 좋은 것)
   - **가벼운 조언:** (취향 차이)
```

이제 터미널에 `/review-code` 라고 치면 저 미친 짓을 한 방에 알아서 다 해줍니다. (또는 그냥 말로 "방금 짠 거 리뷰 좀 해봐" 해도 알아듣고 저 스킬 마법진을 발동합니다).

### 스킬에 인자값 (Arguments) 넘기기

`/버그수정 123` 이라고 치면 깃허브 이슈번호 123번을 고치도록 변수를 넘길 수 있습니다.

```yaml
---
name: fix-issue
disable-model-invocation: true  # 내가 직접 명령어 칠 때만 발동하게 함
---

Github 이슈 #$ARGUMENTS 번호를 해결할 거야.

1. `gh issue view $ARGUMENTS` 명령어로 이슈 본문을 읽고
2. 무슨 에러인지 분석한 뒤
3. 내 코드를 수정해서 고쳐놔 줘.
```

## 레슨 5: 구버전 명령어 (Commands)

이건 Skills 나오기 전에 쓰던 구버전 유물입니다.
`.claude/commands/review.md` 에 대충 프롬프트 하나 적어두면 `/review`로 발동됩니다.

**언제 쓰나요?**
스킬은 디폴트 값 세팅(YAML) 등 좀 세밀하게 조절해야 할 때 쓰고, 그냥 대충 마크다운 파일 딸랑 하나 던져놓고 단축키로 부르고 싶을 땐 Command 쓰면 편합니다.

## 레슨 6: 훅 (Hooks) - 자동화의 극한 달리기

### Hooks 란?

클로드가 어떤 행동을 할 때 앞뒤로 **강제로 낚아채서** 내 코드를 돌리게 만드는 트리거입니다.

### 쓸만한 훅 발동 조건 (Events)

- `PostToolUse` - 클로드가 작업을 끝마쳤을 때 (로직 수정 끝났을 때 자동 포맷팅하기 좋음)
- `PreToolUse` - 클로드가 작업을 저지르기 직전 (파일 지우기 직전에 경고창 띄우기)
- `TaskCompleted` - 거대 태스크가 완료되었을 때

### 실전 예제: 클로드가 파일 수정할 때마다 자동으로 Prettier (코드 예쁘게 정렬) 강제 돌리기

`.claude/settings.json`에 걸어둡니다:

```json
{
  "hooks": {
    "PostToolUse": [
      {
        "matcher": "Write|Edit", 
        "hooks": [
          {
            "type": "command",
            "command": "\"$CLAUDE_PROJECT_DIR\"/.claude/hooks/format.sh"
          }
        ]
      }
    ]
  }
}
```

그리고 `.claude/hooks/format.sh` 스크립트 파일을 만들면:
클로드가 파일을 고친 직후, 저 쉘 스크립트가 빡 실행되면서 코드를 예쁘게 정밀 타격 포맷팅 해버립니다.

## 레슨 7: 커스텀 에이전트(Custom Agents) 맛보기

AI를 특정 성격이나 도구만 쓰도록 복제해서 파생 봇을 만드는 기능입니다.
`.claude/agents/code-reviewer.md` 처럼 만듭니다.

```yaml
---
name: code-reviewer
description: 코드 전문 리뷰 꼰대
tools: Read, Grep
model: sonnet
---
너는 시니어 개발자다. 절대 코드 창조는 하지 말고 읽기(Read)만 해서 평론만 해라.
```

자세한 에이전트 다루는 법은 고급 과정 모듈 21 에서 뼛속까지 파헤칩니다.
