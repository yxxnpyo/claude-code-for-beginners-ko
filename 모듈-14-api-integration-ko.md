## 레슨 4: 거지 같은 인증(Authentication) 통과하기

무방비로 스크립트를 짜면 클로드가 API 키를 하드코딩해버리는 참사가 벌어집니다. 이 부분은 깐깐하게 지시하세요.

### API Key 방식

```
SendGrid 이메일 클라이언트 만들어:
- API 키는 무조건 환경변수(process.env)에서 빼서 쓸 것! (절대 하드코딩 금지)
- 이메일 주소 양식 맞는지 정규식 Validation으로 방어할 것
```

### OAuth 2.0 연동 (고급)

OAuth 연동은 단계가 꼬여있어 짜증 납니다. 클로드에게 통째로 기획을 넘기세요.

```
GitHub OAuth 로그인 플로우 뚫어줘:
1. 클라이언트(유저)를 깃허브 로그인 창으로 토스하는 URL 생성
2. 로그인 끝나고 CallBack 오면 Auth Code 뜯어내기
3. 그 Auth Code를 다시 깃헙에 던져서 진짜 Access Token 받아오는 로직
4. 토큰 안전하게 저장하는 법
5. 나중에 그 토큰 갱신(Refresh) 프로세스까지 싹 다
```

## 레슨 5: 연사 방지 (Rate Limiting)

초당 10번만 쏠 수 있는 API에 100명을 동시에 던지면 사이트가 셧다운(블락) 당합니다. 호출을 줄 세우는 코드를 짜게 하세요.

```
초당 5번만 API 날아가게 API 속도 제한기(Rate Limiter) 클래스 깎아 줘:
- 5번 넘어가면 6번째부터는 큐(Queue)에 세워두고 기다려
- 기다리다가 자기 차례 오면 알아서 발사되게
- 뭔가 에러 나면 일정 시간(지수 백오프) 기하급수적으로 대기 텀 늘리기!
```

## 레슨 6: 통신 에러 핸들링 (Error Handling)

"남의 서버는 언젠가 무조건 터진다"를 전제로 방어 코드를 짜세요.

### 클로드에게 시킬 미친 에러 방어막

```
저 외부 API 쏠 때 겪을 수 있는 모든 경우의 수 에러 핸들링 다 쳐:
- 4xx 에러(클라이언트 잘못 - 예: 400 Bad Request): 이건 우리쪽 잘못이니 두 번 다시 재시도(Retry) 하지 마.
- 5xx 에러(남의 서버 잘못 - 예: 502 Bad Gateway): 이건 쟤네 서버 뻗은 거니까 3번까지 재시도해. 단, 재시도 간격을 1초, 2초, 4초로 (지수 백오프) 점점 늘려가며 쏴!
- Timeout 에러(무응답): 이것도 3번 재시도해.
- 에러 뜨면 유저한테는 "외부 통신 에러입니다" 라고 예쁘게 띄우고, 내 서버 로그파일엔 살벌한 원본 스택트레이스 다 남겨.
```

## 레슨 7: 궁극의 완전판 API Wrapper 깎아내기

위의 모든 레슨을 합쳐서 하나의 무시무시한 지시(Prompt)로 던집니다.

```
스트라이프(Stripe) 결제 API 통신할 완벽한 모듈(Wrapper) 찢어 봐.

핵심 기능:
- 유저(Customer) 가입/수정/삭제
- 카드 결제(Payment) 승인
- 정기구독(Subscription) 등록 및 해지
- 스트라이프 웹훅(Webhook) 리시버

방어 인프라:
- 모든 함수의 입출력 TypeScript Type 완벽 명시
- 커스텀 에러 통제 (4xx, 5xx 구분)
- 500 에러 및 타임아웃 났을 때 자동 재시도(Retry) 및 속도제한
- 두 번 결제 안 되게 멱등성 키(Idempotency keys) 세팅

보안:
- Secret 키는 무조건 환경 변수에서
- 웹훅 날아온 거 진짜 스트라이프 본사에서 쏜 건지 서명 검증(Signature Validation) 하는 로직 꼭 박아 넣어!!
```

## 모듈 14 요점 체크리스트

모듈 15로 배포를 떠나기 전 챙길 것:

- [ ] WebSearch 와 WebFetch 명령어로 낯선 문서를 클로드 뇌에 꽂아 넣을 줄 안다.
- [ ] 단순한 Axios/Fetch 통신을 넘어, 4xx와 5xx 에러 상황을 분리해 재시도(Retry)하는 방어 로직을 생성할 수 있다.
- [ ] API 속도 한계(Rate Limit)를 넘지 않게 큐(Queue)와 대기 로직을 짤 줄 안다.
- [ ] 비밀키를 코드 안에 하드코딩하지 않고 철저히 차단한다.
