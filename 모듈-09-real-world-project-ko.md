# 모듈 9: 프로젝트 실전 - 바닥부터 실제 어플리케이션(Full) 런칭하기

**목표:** 그동안 배운 모든 마술을 총동원해 실제로 프로덕션(실무 배포) 가능한 완벽한 프로젝트 하나 찍어내기

**예상 소요 시간:** 3~6시간 (며칠에 걸려서 느긋하게 부수셔도 좋습니다)

---

## 무엇을 배우게 되나요?

이 모듈은 여태까지 배웠던 모든 지식의 융합체, 엑조디아입니다. 사용자 인증, 데이터베이스 연결, 테스트 코드 작성, 에러 디버깅, 쌈박한 문서작성, 깃(Git) 브랜치 따고 커밋하기, 그리고 실제 배포 구동을 위한 최적화까지 완벽한 "전 과정을 풀 시뮬레이션" 할 겁니다. 덜덜 떨 필요 없습니다, 클로드가 곁에서 모두 코딩해 줄 거니까요.

---

## 튜토리얼 과제 선택 (택 1)

가장 침이 고이는 아이템을 고르세요:

### 옵션 1: 업무 태스크 관리 API 서버 (우리의 메인 튜토리얼 타겟!)

**기능:** 로그인/회원가입, 태스크(할 일) CRUD(쓰고 읽고 지우고), 프로젝트 보드 등
**기술:** Node.js, Express, SQLite, JWT

### 옵션 2: 풀스택 블로그 플랫폼

**기능:** 포스트 작성 수정, 댓글 창, 본문 태그, 검색 등
**기술:** React 기반 화려한 화면 + Node.js 백엔드 조합 등

### 옵션 3: 단축 URL 공장 서비스 (Bitly 짭)

**기능:** 모자란 길이의 커스텀 링크 생성, 클릭 수 트래킹 모니터용, 만료일자 폭파 기능

그 외 날씨 대시보드 옵션 등이 있지만, 우리는 **옵션 1: 일일 태스크 관리 API 빌딩**을 강제로 하드 트레이닝 하겠습니다.

## 페이즈 1: 대규모 공사 기획 및 세팅

### 스텝 1: 기획서 백서(Requirements) 빼오기

아무 생각 없이 `npx` 치면 망합니다. 먼저 클로드에게 기획을 시키세요.

```
나 지금 '태스크 관리 API 앱' 거창하게 만들 거임.
아래 항목들이 꽉 꽉 채워진 프로젝트 요구/기획안 문서 전체를 텍스트로 뽑아와 봐:
- 필수 기능 명세서 리스트업 완벽하게
- 내가 뚫어야 하는 API 엔드포인트 주소 목록 싹 정리
- 데이터베이스 설계(스키마) 그림 정리
- 인증(Authentication)은 뭘로 어떻게 먹일지 구상
```

---

### 스텝 2: 뼈대(Structure) 조감도 그리기

```
방금 네가 내민 '태스크 관리 API' 스펙을 기준으로, 내 폴더 계층 구조와 아키텍처 트리 구조 설계도(routes, controllers, services, models 나눈 거) 예쁘게 제안해 봐.
```

---

### 스텝 3: 프로젝트 빅뱅 (Initialize)

명령을 한 큐에 때립니다.

```
이 '태스크 관리 API'를 구현하기 위한 Node.js 프로젝트의 기초 보일러플레이트 세팅을 내 컴퓨터 터미널에서 당장 쳐내:
- package.json 기본 메타 다 잡아주고
- Express.js 라우터 깔아 놓고
- 데이터베이스 용도 SQLite (better-sqlite3) 설치하고
- 로그인 인증은 JWT, 암호화는 bcrypt 박아놔
- 방어막 테스트 무기인 Jest 깔고
- ESLint랑 Prettier 구동시켜서 코드 예쁘게 정돈해
```

---

### 스텝 4: 폴더 빈 껍데기 대량 생산

```
다음 구조와 완전판 판박이 수준으로 폴더들이랑 텅 빈 파일 껍데기들 터미널에서 전부 한큐에 생성(mkdir & touch) 해버려:
src/ 안에는 routes, controllers, services, models, middleware, utils, config 다 파고,
tests/ 안에는 unit, integration 찢고, 최상단에 .env.example 이랑 .gitignore 도 몽땅 만들어.
```

## 페이즈 2: 심장(데이터베이스) 이식 수술

### 스텝 5: 데이터베이스 테이블 도면 그리기

```
내가 요구하는 스키마 기준대로 SQL 쿼리문들 다 짜서 저장해 놔:

1. Users 테이블: id, 이메일, 패스워드암호화, 이름, 가입일자
2. Projects 테이블: id, 프로젝트명, 소유자_id, 생성일자
3. Tasks 테이블: id, 뭐할지제목, 속한프로젝트_id, 담당자, 중요도, 기한만료일

조건: 외래키(Foreign keys) 칼같이 어긋나지 않게 제약 걸어주고, 조회 속도 쩔게 자주 찾는 칼럼엔 인덱스(Indexes) 다 발라 놔.
```

### 스텝 6: DB 선 연결하기

```
src/config/database.js 파일 하나 멋지게 파서:
- 아까 그 SQLite DB에 선 연결하고 통신 뚫어놓는 로직
- 처음 시작할 때 테이블 없으면 자동으로 아까 그 쿼리문 쫙 돌려주는 마이그레이션 삽입!
```

## 페이즈 3: 입장 컷 (Authentication 유저 인증)

### 스텝 7: 신분증 대조 미들웨어 방어막 구축

```
src/middleware/auth.js 파일에 깡패 미들웨어를 짜 놔:
- 손님이 들고 온 Authorization 헤더에서 JWT 토큰 삥뜯기
- 암호 안 맞거나 위조된 거면 가차 없이 401 빠꾸(Reject) 에러 뱉게 방어 로직 짜
```

### 스텝 8: 회원가입 / 로그인 뚫어주기

```
src/routes/auth.js 라우트에 이 3가지를 뚫어:
[POST] /api/auth/register : 회원 폼 받아서 비밀번호 bcrypt 암호화 해치우고 DB 저장 후 토큰 발급
[POST] /api/auth/login : 아이디 패스워드 대조하고 토큰 주기
```

## 페이즈 4: 진짜 핵심 알맹이 제작 (비즈니스 로직)

### 스텝 9: 프로젝트 보드/태스크 등 CRUD 공장 돌리기

이 타이밍에는 미친 듯이 귀찮은 반복 라우팅을 AI에게 전부 양도합니다.

```
src/routes/projects.js 랑 src/routes/tasks.js 에 각각 다음 짓들을 할 수 있는 라우터 통신 구멍 완성 다 해 놔:

프로젝트용:
- 현재 접속한 유저 지문표상 지가 소유한 프로젝트 리스트만 싹 긁어 읽기(GET)
- 프로젝트 새로 방 파기(POST)
- 수정하고 지우고 CRUD 죄다 (권한 꼭 체크할 것)

태스크용:
- 해당 프로젝트에 할 일 칩 뿌리기(POST)
- 상태 완료(complete) 처리 찍어주기(PATCH)
```

### 스텝 10: 더러운 데이터 거름망 (Validation)

```
유저들이 이상한 특수문자나 숫자 대신 글자 쑤셔넣고 앱 다운 안 시키도록 'express-validator' 패키지를 활용한 1차 입력 검문소 다 만들어서 라우터 앞에 죄다 덕지덕지 수문장 세워 놔.
```


## 페이즈 7: 뒷마무리 및 전시장 청소 (Documentation & Git)

### 깃(Git) 커밋 이쁘게 묶어 발사하기

그냥 `git commit -m "다했어용"` 안됩니다. 의미 있는 단위로 브랜치(Branch) 파서 커밋(Commit) 여러 개로 예쁘게 쪼개라 지시하세요.

```
지금까지 짠 미친 소스코드들을 논리적인 파트별로 끊어서 독립적인 커밋 5개 정도로 예쁘고 자세하게 메세지 자동 생성해서 묶어서 전부 하나하나 커밋 처리(Commit) 해 줘!!
```
