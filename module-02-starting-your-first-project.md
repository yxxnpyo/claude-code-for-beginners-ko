# 모듈 2: 첫 프로젝트 시작하기

**목표:** Claude Code로 개발을 시작하는 다양한 방법 배우기

**예상 소요 시간:** 30-40분

---

## 무엇을 배우게 되나요?

이 모듈에서는 Claude Code로 프로젝트를 시작하는 주요 방법들을 다룹니다. 처음부터 시작하기, 기존 코드 활용하기, 템플릿 사용하기, 저장소 클론하기 등 각 접근법이 언제 적합한지 살펴봅니다.

---

## 레슨 1: 처음부터 시작하기 (가장 일반적인 방법)

### "처음부터"란 무엇을 의미하나요?

빈 디렉토리에서 시작하여 내 의도대로 Claude Code의 도움을 받아 프로젝트를 바닥부터 구축하는 것을 의미합니다.

### 언제 이 방법을 사용하나요?

- 완전히 새로운 것을 만들 때
- 모든 구성 요소가 어떻게 맞물려 있는지 전부 이해하고 싶을 때
- 템플릿으로는 해결할 수 없는 구체적인 요구사항이 있을 때
- 아이디어를 빠르게 프로토타이핑할 때

### 처음부터 시작하는 방법

#### 1단계: 프로젝트 디렉토리 생성

항상 프로젝트를 위한 전용 폴더를 만드는 것부터 시작하세요:

```bash
mkdir my-awesome-project
cd my-awesome-project
```

#### 2단계: Claude Code 시작

```bash
claude
```

#### 3단계: 만들고자 하는 내용 설명하기

구체적이든 포괄적이든 원하는 방식으로 설명하세요:

**간단한 방식:**

```
간단한 투두(todo) 리스트 웹 앱을 만들어줘
```

**더 상세한 방식:**

```
투두 리스트 웹 애플리케이션을 다음 조건으로 만들어줘:
- React 프론트엔드
- Express 백엔드와 REST API
- SQLite 데이터베이스
- 기능: 할 일 추가, 완료, 삭제
- 깔끔하고 모던한 UI
```

처음에는 단순하게 시작하는 것이 좋습니다. 나중에 점진적으로 기능을 추가하는 것이 거대한 첫 프롬프트를 디버깅하는 것보다 훨씬 쉽습니다.

#### 4단계: Claude Code가 구축하는 과정 지켜보기

요청 후 Claude Code는 다음을 수행합니다:

1. 필요시 추가 질문
2. 프로젝트 구조 생성
3. 설정 파일(package.json 등) 세팅
4. 초기 코드 작성
5. 작업 내용 설명

다음과 같은 도구 사용을 관찰해보세요:

- **Write** -- 새 파일 생성
- **Bash** -- `npm init` 등 명령어 실행
- **Read** -- 생성된 내용 확인

#### 5단계: 구조 검토

생성이 끝나면 다음과 같이 물어보세요:

```
프로젝트 구조를 좀 보여줄래?
```

#### 6단계: 테스트하기

```
애플리케이션을 실행해줄래?
```

## 레슨 2: 기존 코드로 작업하기

### 기존 코드베이스란?

오픈소스, 팀의 기존 프로젝트, 레거시 시스템 등 이미 만들어져 있는 코드를 의미합니다.

### 언제 이 방법을 사용하나요?

- 오픈소스에 기여할 때
- 이미 진행 중인 팀 프로젝트에 합류할 때
- 레거시 코드를 유지보수할 때
- 다른 사람의 코드를 보며 학습할 때
- 버그를 디버깅할 때

### 작업 방법

#### 1단계: 프로젝트로 이동

```bash
cd /path/to/existing-project
```

#### 2단계: Claude Code 시작

```bash
claude
```

#### 3단계: 코드베이스 이해하기

탐색을 먼저 요청하세요:

```
이 프로젝트가 어떤 일을 하는지 파악하게 도와줘. 코드베이스를 탐색하고 개요를 설명해줄래?
```

Explore 에이전트를 사용해 주요 파일을 분석하고 스택과 구조를 요약해 줍니다.

#### 4단계: 특정 코드 찾기

```
사용자 인증 로직은 어디에 있어?
```

#### 5단계: 변경사항 만들기

```
API에 사용자 통계를 반환하는 새 엔드포인트를 추가해줘.
```

#### 6단계: 변경사항 테스트

```
기존 기능이 깨지지 않았는지 테스트를 실행해줘.
```

## 레슨 3: 템플릿과 스캐폴딩 사용하기

### 템플릿이란?

기본적인 설정이 모두 완료되어 핵심 로직 구현에 바로 집중할 수 있게 해주는 미리 구성된 프로젝트 구조입니다.

### 일반적인 템플릿

- **create-react-app** 또는 **vite** -- React 앱
- **express-generator** -- Express 서버
- **nest-cli** -- NestJS 앱
- **create-next-app** -- Next.js 앱

### 방법

#### 방법 1: Claude Code에 템플릿 사용 요청

```
create-react-app을 써서 새 React 앱을 만들어줘.
```

#### 방법 2: 커스텀 템플릿 요청

```
TypeScript와 Express로 기본 서버를 만들고 Controllers, Routes, Models 패키지로 나눠줘.
```

#### 방법 3: 프레임워크 CLI 직접 명시

```
TypeScript와 Tailwind CSS를 사용해서 Next.js 프로젝트를 초기화해줘.
```

## 레슨 4: 저장소 클론(Clone) 및 이해하기

### 클론 기능이란?

GitHub 등에서 코드의 복사본을 다운로드하는 것입니다.

### 언제 사용하나요?

- 오픈소스 기여
- 예제 코드 학습
- 튜토리얼 따라가기

### 방법

#### 클론 및 환경 설정

Claude Code 내에서 직접 지시할 수 있습니다:

```
https://github.com/user/awesome-project 레포지토리를 클론하고 셋업을 도와줘.
```

#### 프로젝트 이해

```
이 코드베이스를 파악해서 다음을 설명해줘:
1. 이 프로젝트의 역할은?
2. 사용된 기술 스택은?
3. 구조는 어떻게 되어 있어?
4. 로컬에서 실행하는 방법은?
```

## 레슨 5: 최적의 접근법 선택하기

| 상황 | 최적의 접근법 | 이유 |
|----------|---------------|-----|
| 완전 새로운 프로젝트 | **처음부터** | 모든 과정을 통제하고 학습 가능 |
| 튜토리얼 따라가기 | **클론 후 수정** | 작동하는 코드에서 시작 |
| 표준 웹 앱 | **프레임워크 템플릿 CLI** | 산업 표준 세팅 활용 |
| 오픈소스 참여 | **레포지토리 클론** | 기존 코드 기반 활용 |
| 프레임워크 기초 학습 | **처음부터** | 근본 동작 원리 파악 |
| 매우 빠른 프로토타입 | **템플릿 커스터마이징** | 뼈대 위에 즉시 추가 |

확신이 서지 않을 때 템플릿은 항상 무난한 안전장치이며, '처음부터 만들기'는 당신에게 언제나 최고의 깊이 있는 배움을 선사합니다.

## 자주 묻는 질문 (FAQ)

**Q: 항상 처음부터 만들어야 하나요?**
아닙니다. 일반적인 구조는 템플릿을 써서 시간을 아끼세요.

**Q: 어떤 템플릿을 써야 할지 모르겠어요.**
Claude Code에게 직접 물어보세요. "이런 종류의 앱을 만들려면 어떤 방법으로 시작하는 게 좋아?" 라고 질문하면 됩니다.

**Q: 클론한 코드가 이해가 안 되면요?**
설명해 달라고 요청하세요. Claude Code가 가장 잘하는 일 중 하나입니다.

## 초보자를 위한 핵심 팁

1. **간단하게 시작하고 반복하기** - 뼈대를 먼저 구축한 후 기능들을 점진적으로 더하세요.
2. 보편적인 패턴에는 템플릿을 사용하세요. 바퀴를 다시 발명할 필요는 없습니다.
3. 코드를 수정하기 전에 먼저 읽고 이해하세요.
4. 과정 중간중간 Claude에게 궁금한 것을 질문하고 설명을 요구하세요.
5. README 문서는 언제나 꼼꼼히 체크하세요.
6. Git 사용을 습관화하세요.
7. 자주 테스트하세요. 문제가 눈더미처럼 불어나기 전에 잡아야 합니다.
